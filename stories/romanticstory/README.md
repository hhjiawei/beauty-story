## 【第三部门】剧情策划部 · 节奏工程师（修订版）

**部门描述**：故事的"动态架构师"，基于策划层的核心冲突和人物的性格DNA，定制化开发非线性的段落序列，拒绝套用固定模板。

### 核心工作原则

| 原则 | 说明 | 执行标准 |
|-----|------|---------|
| **因人设戏** | 每段内容必须源于人物的**性格缺陷**和**核心恐惧** | 对照人物部DNA模板，标注驱动源 |
| **因事设段** | 段落划分由**冲突事件**决定，非字数均分 | 每段解决一个具体问题或制造一个新问题 |
| **关系为轴** | 所有事件必须**改变或揭示**人物关系 | 禁止纯功能性事件（如单纯展示职业） |
| **弹性结构** | 支持线性/闪回/多线并行/环形等任意结构 | 结构选择必须服务于情绪目标 |

### 核心任务清单（非线性流程）

#### 阶段一：冲突拆解（基于输入，非凭空创造）

| 步骤 | 任务 | 输出 | 检查点 |
|-----|------|------|--------|
| 1.1 | 将主矛盾拆解为**3-5个不可调和的具体冲突** | 冲突清单（每个≤30字） | 每个冲突必须涉及双方，非单方困境 |
| 1.2 | 为每个冲突匹配**触发人物缺陷的触发器** | 缺陷-冲突映射表 | 标注哪个人物的哪个缺陷被触发 |
| 1.3 | 设计冲突的**升级机制**（非重复，是质变） | 升级路径图 | 每次升级必须改变权力关系 |
| 1.4 | 确定冲突的**空间载体**（在哪里发生） | 场景-冲突匹配表 | 空间必须加剧冲突（如：电梯困住逃避者）|

**冲突升级类型库**（参考，非强制）：
- **信息升级**：从猜测→部分知情→完全知情→知情但无法行动
- **关系升级**：从陌生→被迫绑定→自愿靠近→主动撕裂→重构
- ** stakes升级**：从面子→利益→秘密→尊严→生存/存在
- **空间升级**：从公共→半公共→私密→困锁→逃离

#### 阶段二：段落生成（基于冲突，非基于字数）

**段落定义**：一个**完整的变化单元**，包含：
- 一个**触发事件**（外部或内部）
- 一次**人物反应**（必须符合性格DNA）
- 一个**关系变化**（从状态X到状态Y）
- 一个**遗留问题**（驱动下一段）

**段落生成公式**：
```
当前关系状态 
+ 人物A的缺陷驱动行为 
+ 人物B的缺陷驱动反应 
+ 空间/时间的压力 
= 冲突爆发 
→ 新关系状态 
+ 新遗留问题
```

### 与固定分幕的核心区别

| 维度 | 固定分幕（旧） | 动态生成（新） |
|-----|-------------|--------------|
| **划分依据** | 字数均分（1500/2000/1500/1000） | 冲突事件的自然解决/爆发点 |
| **结构来源** | 预设模板（起承转合） | 人物缺陷的碰撞模式 |
| **段落数量** | 固定4幕×3-5场景=12-20段 | 弹性5-8段（由冲突复杂度决定）|
| **时间处理** | 线性推进 | 支持闪回、压缩、延展 |
| **空间处理** | 场景切换自由 | 空间必须服务于冲突加剧 |
| **关系推进** | 每幕笼统标注 | 每段精确标注从X到Y |
| **相识/职业** | 可能在前幕展开 | 全程压缩，无独立段落 |

### 【第四部门】共情点设计部 · 情绪炼金师

**部门描述**：故事的"情绪曲线设计师"，在关键节点植入情感爆点，确保读者情绪被精准操控。

**核心任务清单**：

| 序号 | 任务项 | 输出要求 | 量化标准 |
|-----|-------|---------|---------|
| 1 | **情绪曲线图** | 标注全文情绪指数（-10到+10）| 每500字一个节点，必须呈W型或心跳型 |
| 2 | **共情锚点设计** | 设计8-12个共情锚点 | 每锚点包含：触发器+躯体反应+反差设计 |
| 3 | **金句预埋** | 输出3-5个高光台词 | 台词必须**潜台词丰富**，可截图传播 |
| 4 | **五感清单** | 为每个关键场景分配**主导感官** | 每场景至少2种感官，避免单一视觉 |
| 5 | **反差设计** | 设计3处**表面与内在的强烈反差** | 如：表面的不在乎 vs 内心的深爱 |
| 6 | **留白设计** | 标注2-3处**情绪留白点**（不写满）| 留白用于让读者自行投射 |

### 【第五部门】分块写作部 · 主笔作家

**部门描述**：故事的"执行导演"，将施工图转化为文字，确保每段推动关系且充满人味。

**核心任务清单**：

| 序号 | 任务项 | 执行标准 | 检查方式 |
|-----|-------|---------|---------|
| 1 | **场景化写作** | 用动作和环境体现心理，禁止直接写心理活动 | 删除所有"她想"、"他感到"类句子，看是否仍通顺 |
| 2 | **关系推进执行** | 每段必须有**关系状态变化**或**关系信息量释放** | 对照剧情部的关系变化表检查 |
| 3 | **相识过程压缩** | 如涉及相识，用**一句话背景交代**，禁止场景展开 | 全文搜索相识相关段落，确保≤200字 |
| 4 | **职业标签化** | 职业仅作为身份标识，禁止工作细节描写 | 全文搜索职业相关段落，确保无流程、技能、专业术语展开 |
| 5 | **对话设计** | 每句对话必须**推动剧情**或**揭示性格**，有潜台词 | 删除对话，看剧情是否断裂 |
| 6 | **钩子植入** | 每段结尾必须有**小钩子**（悬念/危机/发现）| 检查每段最后一句是否驱动续读 |
| 7 | **限制视角** | 单段内**锁定单一视角**，禁止随意切换 | 每段标注视角角色，检查一致性 |
| 8 | **身体叙事** | 用**躯体反应**代替情绪标签 | 检查是否有"她很伤心"类标签，改为躯体反应 |

**写作规范检查表**：

| 检查项 | 合格标准 | 不合格示例 | 合格示例 |
|-------|---------|-----------|---------|
| 相识处理 | ≤200字，无场景 | 详写如何相遇、如何心动 | "他们三年前在一场乌龙相亲中认识，此后互不联系" |
| 职业处理 | 仅标签，无细节 | 写具体工作流程、专业技能 | "他是律师"（仅此而已，不写出庭细节）|
| 心理描写 | 用动作体现 | "她感到很难过" | "她把纸巾捏成一团，又展开，又捏团" |
| 对话功能 | 每句有推动或揭示 | "你吃了吗""吃了"（水对话）| "你吃了吗"（实际想问：你还在生气吗？）|
| 视角统一 | 单段单一视角 | 前句写A心理，后句写B心理 | 全段锁定A的所见所感 |
| 段落钩子 | 结尾驱动续读 | "然后他们回家了"（闭环）| "然后她看到了门口的鞋——那双她再熟悉不过的鞋" |

**自检清单（写作中）**：
- [ ] 这段如果删掉，**人物关系**还成立吗？（如不成立，留下；如成立，删掉）
- [ ] 这段是否包含**相识前史展开**？（如有，压缩至一句话）
- [ ] 这段是否包含**职业工作细节**？（如有，删除）
- [ ] 这段的**视角角色**是谁？是否全程锁定？
- [ ] 这段结尾是否有**续读钩子**？

### 【第七部门】去AI味处理部 · 文字调音师

**部门描述**：故事的"化妆师"，消除机械感，增加人味与生活毛边，确保文本有温度。

**核心任务清单**：

| 序号 | 任务项 | 处理方法 | 检查标准 |
|-----|-------|---------|---------|
| 1 | **打破对称** | 打散排比句，长短句交错，加入口语碎片 | 无连续3句结构相似 |
| 2 | **删减逻辑词** | 删除"因为/所以/然而"等因果连接词，用动作暗示 | 每500字逻辑词≤3个 |
| 3 | **词汇降维** | 将书面语改为口语，加入方言/行业黑话 | 无"父亲/用餐"类正式词 |
| 4 | **消除标签** | 删除情绪标签，改为躯体反应 | 无"她很伤心"类句子 |
| 5 | **限制视角强化** | 确保视角受限，只写该角色能感知的 | 无全知描述 |
| 6 | **语法破坏** | 适当使用省略句、断续句、倒装句 | 有10%句子不完全合规 |
| 7 | **具体化扫描** | 将抽象概括改为具体时间/物品/动作 | 无"很久/很多东西"类词 |
| 8 | **人味添加** | 加入不完美（口误、走神、小动作）| 每段至少有1个小瑕疵 |

**AI味特征库与修正方案**：

| AI味特征 | 识别标准 | 修正方案 | 示例 |
|---------|---------|---------|------|
| 过度对称 | 连续3句以上结构工整 | 打乱结构，加入碎片 | "他来了。他走了。他忘了。"→"他来了。又走了。忘得一干二净。" |
| 逻辑过剩 | 因果连接词频繁 | 删除因果词，用动作 | "因为下雨，所以迟到"→"雨大，他浑身湿透地推门，时钟指向九点零三" |
| 词汇洁癖 | 用词正式，无烟火气 | 改为口语/方言 | "父亲"→"爸"，"用餐"→"吃饭" |
| 情感标签 | 直接写情绪状态 | 改为躯体反应 | "她很伤心"→"她把纸巾捏成一团" |
| 全知视角 | 随意切换内心OS | 锁定视角 | 删除"他不知道的是..."类句子 |
| 完美语法 | 每句主谓宾齐全 | 适当省略 | "他走了。"→"走了。" |
| 抽象概括 | 时间/动作模糊 | 具体化 | "很久"→"四十分钟"，"很多东西"→"三页纸" |

**润色工序**：
1. **对称打破**：扫描排比句，打乱节奏
2. **逻辑词删减**：高亮"因为/所以/然而/但是"，删除或替换
3. **词汇降维**：替换书面语为口语
4. **标签消除**：高亮情绪词，改为躯体反应
5. **视角锁定**：检查每段视角，删除全知描述
6. **语法破坏**：选择10%句子进行省略/倒装处理
7. **具体化**：替换所有抽象时间/数量词
8. **人味添加**：每段加入1个小瑕疵（口误、小动作、走神）

**最终测试**：
- 随机抽取3个段落，混入3个人工写作段落
- 测试读者无法准确识别AI段落→通过
- 识别出AI段落→分析原因，二次润色

请基于class CharacterBasic(TypedDict):
    name: str
    age: int
    sample_character: str # 身高外貌（突出 1 个标志性特征，如眼尾痣、指尖薄茧）
    career_tag: str  # 身份职业：职场人 / 学生 / 自由职业者（贴合背景，决定行为逻辑）
    income_level: str # 普通 / 优渥 / 清贫（影响爱情观、现实顾虑）
    habit: str  # 小癖好（喝咖啡加糖、戴旧手表、习惯性低头）

    """
        性格特质：主性格 + 反差感（如外冷内热、嘴硬心软、温柔敏感）
        原生家庭：影响爱情观的关键（缺爱→渴望安全感，强势→不懂表达）
        核心执念 / 软肋：情感痛点（怕被抛弃、不敢告白、放不下过去）
        爱情观：对感情的态度（慢热、勇敢、现实、被动）
        成长弧光：短篇需小弧光（从胆怯到勇敢，从冷漠到温柔）
        核心动机：行为驱动力（想被爱、想守护、想弥补遗憾）
    """
class CharacterDNA(TypedDict):
    surface_personality: List[str]  # 外在表现（3个关键词）
    inner_essence: str  # 内在本质（核心恐惧+核心渴望）
    character_flaw: str  # 性格缺陷（驱动冲突的关键，如：逃避型依恋） !! 重点性格描述
    characteristics: str  # 性格缺陷（驱动冲突的关键，如：逃避型依恋） !! 重点性格描述  性格特质：主性格 + 反差感（如外冷内热、嘴硬心软、温柔敏感）
    core_mechanism: str  # 核心动机：行为驱动力（想被爱、想守护、想弥补遗憾）

class RelationshipDynamics(TypedDict):
    initial_state: str # 初始关系状态
    turning_points: List[str]  # 每章的发展状态
    final_state: str    # 最终关系状态

class CharacterProfile(TypedDict):
    character_id: str
    basic: CharacterBasic
    dna: CharacterDNA
    relationship_dynamics: RelationshipDynamics
    physical_markers: List[str]  # 身体特征状态

class NetworkNode(TypedDict):
    from_char: str
    to_char: str
    relationship: str       # 关系
    emotional_current: str # 情感流向（如：A恨B但放不下）
    motivation: NotRequired[str]  # 仅配角需要
    independence: NotRequired[str]  # 仅助攻者需要

class CharacterState(TypedDict):
    """人物组输出状态"""
    characters: List[CharacterProfile]    # 主角配角等人物属性
    network: List[NetworkNode]  # 关系网络图  和要求 **部门描述**：角色的"造物主"，设计完整人物档案与关系网络，为情感张力提供基础。
