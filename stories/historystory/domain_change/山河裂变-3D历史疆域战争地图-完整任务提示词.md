# 山河裂变 — 3D 中国历史疆域战争地图 · 完整任务提示词

你是一名世界级的 3D 图形工程师与数据可视化专家，精通 Three.js、自定义 GLSL 着色器、GeoJSON 地理数据管线与实时动画编排。请构建一个完整的、具备生产质量的全屏网站：在真实 DEM 地形上渲染一幅 3D 中国地图，依据 `events.json` 中的战争事件，驱动各势力疆域随时间动态变化——占领、灭国、联盟、反叛、归附，每一种事件都有专属动画；都城之上立有写着势力名的 3D 战旗。

最终结果必须是下述规格的忠实功能性实现，而不是一个泛泛的"地图演示"。要实现列出的每一个系统、控件、动画、响应式行为、加载状态、错误状态与性能保障。不要使用占位代码、静态截图、预渲染视频、正方形/人造多边形疆域，或任何简化版替代品。

## 2. 数据契约

### 2.1 输入：cities.json（已有样例，直接使用）

```json
[{ "id": "bo", "name": "亳", "location": [115.65, 34.45], "owner": "shang", "tier": 1, "note": "商汤之都" }]
```

- `location` 为 `[lon, lat]` WGS84；`tier`: 1=都城，2=重要城邑，3=普通城邑/关隘。
- 预处理为每城补充 `county_idx`（所在县面索引），写入输出 `cities.json`。

### 2.2 输入：factions.json（已有样例，直接使用）

```json
[{ "id": "shang", "name": "商", "color": "#a63a2b", "founded": -1600, "fallen": null }]
```

- 预处理为每势力预计算 `vassal_color`（主色向白色提亮 30%，供联盟/附庸显示）与 `stroke`（主色压暗 25%），写入输出 `factions.json`。

### 2.3 输入：events.json（已有样例，直接使用）

事件对象字段：`id, seq, year, yearLabel, title, category, type, actors{primary,target,supporters}, routes[], battle{}, outcomes[], summary, quote{text,source}, camera{zoom,pitch}?`。

- `type` 取值：`proclaim | conquer | battle | march | occupy | ally | ally_break | submit | rebel | move_capital | succeed | other`。
- 只处理引起疆域变化的事件类型：**占领 occupy、灭国 conquer、联盟 ally、反叛 rebel、归附 submit**（灭国由 `outcomes[].type=="conquer"` 驱动，可能出现在 `battle` 事件内）。其余类型（proclaim / move_capital / succeed / battle / march / other）只触发仪式动画与镜头，不改疆域。
- `routes[]`：`{faction, from:城市id, toCity:城市id|null, toPoint:[lon,lat]|null, waypoints[]}`。
- `outcomes[]` 驱动状态机，语义如下：
  - `{type:"occupy", city, to}` → 该城归 `to`（**占领**）
  - `{type:"conquer", faction, by}` → `faction` 的全部城市归 `by`，`faction` 灭亡（**灭国**）
  - `{type:"ally", leader, members[]}` → members 成为 `leader` 的附庸：`vassal_of=leader`，颜色变为 leader 主色的浅色系，owner 不变（**联盟**）
  - `{type:"ally_break", leader, members[]}` → 解除附庸，恢复原色
  - `{type:"submit", faction, to}` → `faction` 的全部城市归 `to`（**归附**）
  - `{type:"rebel", faction, to, cities[]}` → 所列城市倒戈归 `to`（**反叛**）
  - `{type:"move_capital", faction, from, to}` → 都城迁移（旗帜迁移）
  - `{type:"proclaim", faction, ruler, title}` / `{type:"ruler_change", ...}` → 仪式事件

### 2.4 输出：timeline.json（预处理后端对端契约）

```json
{
  "initial": {
    "year": -1600,
    "states": [{ "county_idx": 0, "owner": "shang", "vassal_of": null }],
    "capitals": { "shang": "bo", "xia": "zhenxun" },
    "fallen": []
  },
  "events": [
    {
      "id": "shang-0004", "seq": 4, "year": -1592, "yearLabel": "约前1592年",
      "title": "鸣条之战", "type": "battle",
      "summary": "……", "quote": { "text": "……", "source": "《史记·殷本纪》" },
      "actors": { "primary": "shang", "target": "xia", "supporters": [] },
      "routes": [ { "faction": "shang", "from": "bo", "toCity": null, "toPoint": [111.05, 35.05], "waypoints": [] } ],
      "battle": { "name": "鸣条", "location": [111.05, 35.05], "note": "今山西运城夏县西" },
      "camera": { "zoom": 7.4, "pitch": 58 },
      "changes": [{ "county_idx": 812, "from": "xia", "to": "shang", "vassal_of": null }],
      "fallen": ["xia", "kunwu"],
      "vassals_set": [], "vassals_clear": [],
      "capital_moves": []
    }
  ]
}
```

`changes` 是该事件引起的**县级归属增量**（仅列出变化的县），前端只应用增量。疆域五型动画全部由 `changes + fallen + vassals_set/clear` 驱动。

### 2.5 输出：counties.bin（预三角化县面网格）

二进制布局（小端）：头部 JSON 长度 `uint32` + 头部 JSON（`{count, bbox}`）+ `positions: Float32Array`（经度,纬度,0 三元组）+ `county_idx: Uint16Array`（每顶点一个）。县界折线 `county_lines.json` 为 `[{rings: [[lon,lat],...]}]`。浏览器端零三角化、秒开。

## 3. 离线预处理管线（tools/preprocess.py）

Python 3.11+，依赖 `geopandas shapely pandas numpy pillow mapbox-earcut`。用 `asyncio` 无必要，纯顺序执行即可。五个阶段：

### 阶段 0：加载与索引县面

读取 `data_raw/geoBoundaries-CHN-ADM3.geojson`，确保 `EPSG:4326`，按行序赋 `county_idx`（0..N-1）。保留 `shapeName` 仅用于调试。

### 阶段 1：城池 → 县面空间关联

以 `predicate="within"` 做 `gpd.sjoin`。关联失败的城市（坐标落在县面外，如示意位置）：回退为"最近县面质心 ≤ 0.5°"则绑定，否则记 `county_idx=-1` 并写入 `failed_cities.json` 警告，**不得静默丢弃**。

### 阶段 1.5：neutral 晕染（解决资料不全的空白县）

县面无城池即 `neutral`。对每个 neutral 县，计算其质心到全部控制点（有 `county_idx` 的城池）的距离，归属**最近控制点的 owner**（含其 `vassal_of`），距离上限 `0.8°`（约 80km），超出则保持 neutral。此步骤在**初始状态**与**每个事件应用后**各执行一次（复用参考管线 `fill_neutral_counties` 的逻辑与参数）。neutral 在前端以接近透明渲染。

### 阶段 2：事件回放，生成 timeline.json

按 `seq` 排序回放 outcomes，维护 `cities_state{owner, vassal_of, last_event_time}` 与 `counties_state`。一县多城时按优先级裁定归属：`都城4 > 郡治3 > 县治2 > 关隘1 > 要塞0`，并列时非附庸优先、再按最近事件时间。每事件输出县级增量 `changes`（对比前后状态，只记变化县）、`fallen`、`vassals_set/clear`、`capital_moves`。

### 阶段 3：势力外轮廓（boundaries/{event_id}.json）

对初始状态与每个事件后状态：县面按 `owner` `dissolve`，然后**边界平滑**——`buffer(+0.008°)` → `simplify(0.005, preserve_topology=True)` → `buffer(-0.0048°)`；删除面积 `< 0.001°²` 的碎片；提取外环折线，按 `[{faction, rings:[[lon,lat],...]}]` 写出。线条坐标保留 4 位小数。

### 阶段 4：网格与纹理烘焙

1. **heightmap.png + terrain.json**：拼接 `china_dem` zoom=6 覆盖中国范围（lon 73–135.1，lat 17.9–53.6）的瓦片，重投影为**等经纬（Plate Carrée）网格**，输出 4096×2304 左右 PNG。高程编码支持 terrain-rgb（`h = -10000 + (R*65536+G*256+B)*0.1`），若实际瓦片为其它编码（terrarium/灰度），在脚本顶部用 `DEM_ENCODING` 配置项切换。`terrain.json` 记录 `{bbox:[minLon,minLat,maxLon,maxLat], min, max, encoding}`。
2. **relief.jpg**：同网格同像素尺寸，拼接 `china_relief`（或 china_dem 同源渲染图）zoom=6 瓦片，质量 85。
3. **counties.bin / county_lines.json**：县面 `simplify(0.001°, preserve_topology=True)` 后用 earcut 三角化（含孔洞），按 §2.5 写出；县界折线同步简化。

管线结束打印验收摘要：县总数、城池关联成功率、每事件变化县数、各文件体积。任何阶段失败必须给出中文可读错误。

## 4. 前端技术架构

### 4.1 坐标系与投影

统一使用等经纬投影，世界单位 = 公里：

- 中央基准 `lon0=104, lat0=36`；
- `x = (lon - lon0) * 111.32 * cos(36°)`，`z = -(lat - lat0) * 110.94`；
- 高度 `y = elevation_km * exaggeration`，`exaggeration` 为全局 uniform（默认 1.8，UI 可调 1.0–3.0）。

所有地理图层（地形、县面、河流、城市、旗帜、路线）共用此投影与同一个 `heightmap` 采样函数，保证完美贴合（draping）。

### 4.2 渲染管线

1. `THREE.WebGLRenderer`：`antialias:true`、`powerPreference:'high-performance'`、`pixelRatio = min(devicePixelRatio, 2)`、`ACESFilmicToneMapping`、曝光 1.05、场景背景 `#0b0d10`。
2. 透视相机 FOV 45，near 1，far 20000。
3. `EffectComposer`：RenderPass + `UnrealBloomPass`（强度 0.35、半径 0.5、阈值 0.85）+ OutputPass。
4. `OrbitControls`：阻尼 0.08，`minDistance 120 / maxDistance 4200`，`maxPolarAngle = π*0.49`，目标 `(0, 0, 0)`，初始自动旋转速度 0.15，可开关。
5. 光照：`DirectionalLight`（太阳，颜色强度由"时辰"系统驱动，见 §9）+ `HemisphereLight`（天光 `#8fa3c0` / 地光 `#2a241c`，0.55）。

### 4.3 场景图层（自底向上）

`terrain → ocean/water → territory fills → county lines → rivers/lakes → faction boundaries → route arrows → cities → flags → battle FX → labels(CSS2D)`。

## 5. 地形系统

1. **网格**：覆盖 `terrain.json` bbox 的 `PlaneGeometry(宽度km, 高度km, 512, 288)`，旋转平铺于 XZ。
2. **顶点着色器**：采样 `heightmap.png`（线性过滤）得高程，`y = h_km * uExaggeration`。高程 texture 与县面填充共用同一采样函数。
3. **法线**：片元阶段对 heightmap 做有限差分（纹素偏移 1px）生成细节法线，随 `uExaggeration` 缩放——**调大夸张系数时山体质感必须同步增强**。
4. **配色**：以 `relief.jpg` 为底，叠加分层设色（海拔 0–500m 低地暖褐绿、500–1500m 黄土、1500–3000m 灰褐、>3000m 灰白）与坡度晕渲（`dot(N, sunDir)`），整体压暗 20% 以衬托疆域色块。山脊线处轻微提亮。
5. **边缘**：bbox 外圈用一圈向下延伸的"崖壁"裙边几何收边，配色 `#0b0d10`，杜绝看到地形薄片截面。
6. **海洋**：`ocean.json` 渲染为 `y=0` 的平面，深蓝黑 `#0d1b24`，片元叠加轻微 FBM 微光（复用 §10 噪声栈）；`coastline.json` 画 1px 柔光海岸线。

## 6. 疆域渲染系统（核心）

### 6.1 县面填充

- 加载 `counties.bin` 构建一个合并 `BufferGeometry`；每顶点带 `aCountyIdx` 属性。顶点着色器采样 heightmap 位移（`y = h*uExaggeration + 0.9km` 抬升防 z-fight，并开启 `polygonOffset(-1,-1)`）。
- **状态纹理**：两张 `DataTexture`（64×64 RGBA8，覆盖 4096 县槽位；超出则扩到 128×32…按实际县数取最小 2 的幂）——`uStatePrev` 与 `uStateNext`，RGB 为该县当前显示色（附庸浅色已预计算）、A 为归属标志（255=有主，60=neutral）。
- 顶点着色器用 `aCountyIdx` 换算纹素 UV 取色，`mix(prev, next, localT)` 输出 varying 至片元；`localT = smoothstep(delay, delay+0.35, uProgress)`，`delay` 为每顶点属性（用于级联动画，见 §8）。
- neutral 县：填充 `#1a1f26`、不透明度 0.18，仅县界线隐约可见。
- 被灭势力原色从 `factions.json` 移除后，其残县自然随 `changes` 变色，不留残色。

### 6.2 县界线

`county_lines.json` 合并为一份 `LineSegments`，顶点同样采样地形位移（抬升 1.0km），颜色 `#0e1116`、不透明度 0.35。随相机距离调节不透明度（>2500km 时淡出至 0.1，避免远景观感脏乱）。

### 6.3 势力外轮廓（边界"长城线"）

- 加载 `boundaries/`：每事件一套 `{faction: rings}`。渲染为抬升 1.6km 的发光折线（`Line2`/`LineMaterial`，线宽 2.5px，势力主色，additive 混合，bloom 拾取发光）。
- 事件切换时：旧轮廓 0.6s 淡出、新轮廓 0.6s 淡入（交叉淡化，不做顶点 morph——允许轮廓"重画"感，配合疆域变色已足够连贯）。
- 附庸领地按"同色系势力圈"处理（预处理 dissolve 时按 `vassal_of ?? owner` 分组），区分两种空间情形：
  - 与宗主领地**地理连片**时：共用一条宗主色实线外轮廓，附庸一侧不另起线，视觉上浑然一体；
  - 与宗主**远隔（飞地）**时：dissolve 自然产生独立环，该环以**宗主色虚线**（dash 4km / gap 3km）、线宽 1.5px 绘制——与宗主本土的实线圈一眼可辨"此乃附庸飞地，非本土"；附庸填充仍为其浅色系，归属色感不丢失；
  - 联盟光链（§8.2）始终连接宗主与远隔附庸的都城，距离再远也能读出其从属关系。

### 6.4 河流与湖泊

`rivers.json`：淡青蓝 `#7fa8c9` 细线（1px），顶点采样地形（抬升 0.5km，高程下限 2m），不透明度 0.5；`lakes.json`：填充 `#16303e`、不透明度 0.8。黄河、长江干流可在线宽上加倍——按要素 `name` 字段匹配 "Yellow/Yangtze/黄河/长江"。

## 7. 城市、都城与 3D 旗帜系统

### 7.1 城市标记

- tier1 都城：三层同心圆环 + 立柱微光，直径 8km；tier2：实心圆点 5km；tier3：圆点 3km。颜色 = 当前归属势力主色，附庸用浅色。几何：低分段 `CircleGeometry`/`RingGeometry` 实例化（`InstancedMesh`），顶点着色器贴地。
- 归属变化时城市标记颜色随 `uProgress` 同步插值。
- 标签：CSS2D，Noto Serif SC。tier1 常显（14px 加阴影），tier2 相机 <1800km 显示（12px），tier3 <700km 显示（10px，0.7 透明度）。标签避让不强制，允许自然重叠淡出。

### 7.2 3D 战旗（每势力一面，立于都城）

- **旗杆**：`CylinderGeometry(0.4km, 0.6km, 26km)`，深玄色 `#2b2620`，顶端宝珠（`SphereGeometry`）鎏金 `#c9a45c`。
- **旗面**：`PlaneGeometry(13km, 8.5km, 24, 16)`，顶部起于杆顶下 1km。自定义 `ShaderMaterial`：顶点波动 `z += sin(x*0.55 - uTime*2.6) * (0.5 + 0.5*x/13) * 0.9km` + 高频细节 `sin(x*1.7 - uTime*4.1)*0.25`，根部固定（x=0 处振幅 0）。
- **旗面纹理**：Canvas 512×336 程序化绘制——势力主色打底，四周回纹/素边深色镶边（压暗 25%），中央势力名大字（`700 240px "Noto Serif SC"`，宣纸白 `#f2ead8`，竖排单字如"夏""商""周""秦"），轻微做旧噪点。绘制前 `await document.fonts.load(...)`。纹理 `anisotropy = renderer.capabilities.getMaxAnisotropy()`。
- 旗帜随地形高度立于都城坐标；**双面渲染**。
- 动画原语：`flagRise`（建旗：旗面自杆底 1.2s 攀升至顶 + 展开缩放 0→1 + 金色脉冲环）、`flagFall`（灭国：旗面 1s 内下垂旋转 25°、透明度→0、整旗沉入地形）、`flagMove`（迁都：旧都旗落同时新都旗起，镜头跟随）。

### 7.3 战场标记

`battle.location` 处：交叉双剑用几何体拼不出就用——交叉的两枚扁平箭头锥体（`ConeGeometry` 旋转交叉），暗红 `#8c2f22`，常显直至事件结束；事件触发时叠加冲击波特效（§8）。

## 8. 事件引擎与动画编排

### 8.1 时间轴引擎

- 加载 `timeline.json` → 应用 `initial` 状态（写 `uStatePrev=uStateNext`）。
- 播放器状态机：`idle | playing | animating`。`playing` 时每事件停留 ` dwell = 2.2s / speed`，随后进入 `animating`。
- 速度档位 0.5 / 1 / 2 / 4×，默认 1×。
- 每事件执行序列：`镜头飞行(0.9s, easeInOutCubic) → 类型动画(时长见下) → 更新 uStatePrev=uStateNext、交换纹理 → dwell`。
- `changes` 落盘到状态纹理：CPU 更新 `uStateNext` 像素，`uProgress` 0→1 播放；`delay` 属性按县质心到进攻方都城的距离分档（`delay = dist/2600km * 0.6s`），形成**自东向西/由近及远的颜色级联**——灭国时尤为壮观。

### 8.2 六类动画

| 事件 | 时长 | 编排 |
| --- | --- | --- |
| **占领 occupy** | 2.6s | 行军箭头沿 route 推进（§9）→ 目标城金色闪光 → 该县色块级联变为新主色 → 城标变色 |
| **灭国 conquer** | 4.2s | 进攻方全军箭头 → 全部失县按距离级联变色 → 亡国旗 `flagFall` → 其图鉴置灰加"亡" → 战场冲击波（若带 battle） |
| **联盟 ally** | 2.2s | 成员县颜色过渡为宗主浅色系 → 宗主都城与各成员都城之间升起弧形光链（三次贝塞尔，顶点贴地+高度弧，0.8s 生长）→ 光链常驻至 ally_break |
| **反叛 rebel** | 2.4s | 倒戈县红色脉冲闪烁 3 次（0.9s）→ 倒戈箭头 → 色块转为目标色 → 城标变色 |
| **归附 submit** | 3.0s | 该势力全部县 0.4s 白闪 → 级联转为目标色 → 都城 `flagFall` 同时目标方该城 `flagRise`（旗面文字换为目标势力名） |
| **仪式类** proclaim/move_capital/succeed/battle/march/other | 1.8–3.5s | proclaim：都城 `flagRise` + 金环扩散；move_capital：`flagMove`；battle：行军箭头对冲 + 战场冲击波 + 镜头微震（0.15s 振幅 2km）；march：仅行军箭头；succeed/other：都城金色脉冲 + 事件卡 |

### 8.3 事件卡与引文

每事件进入时，左下角事件卡（玻璃拟态）淡入：`type` 徽章（占领=赤/灭国=玄/联盟=金/反叛=绛/归附=青/仪式=灰）、`yearLabel + title`、`summary`、`quote.text` + `quote.source`（楷体感、前置「」）。动画结束 1.2s 后淡出；用户暂停时常驻。

## 9. 行军路线与特效

1. **路线几何**：`from`→(`waypoints`)→`toCity|toPoint`，球面大圆插值 64 段，顶点着色器贴地抬升 2.2km。
2. **箭头动画**：`Line2` 虚线（dashSize 6km gap 4km），`uProgress` 控制出鞘生长；头部一枚 `ConeGeometry` 箭头沿路径切线移动；颜色 = 行军势力主色，additive，bloom 泛光。
3. **对冲**：双方 route 同屏时，两箭头相遇于 battle.location 触发冲击波——`RingGeometry` 扩散（0→90km，1.1s，easeOut）+ 顶点抬起的尘埃 `Points`（200 粒，重力回落，1.6s 生命）+ 0.15s 镜头微震。
4. **时辰系统（光照滑块）**：输入 `t∈[0,100]` 默认 62。方位角 `lerp(-1.0, 1.0)`、高度角 `lerp(0.10, 0.62)`；太阳色由黄昏 `(1.0,0.55,0.30)` 到正午 `(1.0,0.95,0.85)` 插值，强度 1.2–2.0。标签：<20 清晨 / <45 上午 / <75 午后 / 否则 黄昏。全部光照 uniform 连续过渡。

## 10. 程序化噪声栈（复用）

GLSL 函数库：`hash2(p)`（`fract(sin(dot(p,(127.1,311.7)))*43758.5453)`）、`gradNoise(p)`（五次插值 `f³(f(f·6-15)+10)`）、`fbm(p)`（3 个八度，振幅 1/0.5/0.25，频率 1/2.04/4.11）。用于：海洋微光、旗帜做旧、尘埃粒子扰动。禁止贴图噪声。

## 13. 加载与失败状态

- **加载屏**：`#0b0d10` 全屏；眉题 `Three.js · WebGL2 · GeoJSON`；大标题 `山河裂变`（宽字距）；168×2px 加载轨道内金色光条循环；状态文字依次为 `加载地形… → 烘焙疆域… → 竖立战旗…`。首个成功渲染帧后 0.9s 淡出并禁用指针。
- **数据失败**：任一 `data/` 文件 404/解析失败 → 加载屏转错误态：标题 `数据加载失败`，列出缺失文件与「请先运行 tools/preprocess.py 生成数据」的提示，暖珊瑚色标题。
- **WebGL2 不可用**：`WebGL 不可用` + 推荐使用最新版 Chrome / Edge / Firefox 的说明。
- 所有初始化包在 try/catch 中，原始错误打印到 console，界面显示可读摘要。

## 14. 动画循环与生命周期

`renderer.setAnimationLoop(frame)`。每帧：取高分辨率时间 → delta 钳制 ≤0.1s → 推进 `uTime` → 更新 OrbitControls → 事件引擎 tick → composer.render() → 每 0.5s 刷新 FPS → 首帧后揭示 UI。`document.hidden` 时暂停，恢复时重置时间戳防跳变。resize 时更新相机宽高比与渲染器尺寸（保留 DPR 上限）。滑块输入只改 uniform，**不得重建几何体、材质或后期通道**。每帧零对象分配（复用向量/矩阵临时对象）。

## 16. 质量与实现约束

- 疆域边界必须来自 geoBoundaries 县面几何（经平滑），**禁止**正方形、正多边形、手绘感多边形。
- 中立空白县必须经晕染处理，不得出现突兀的"资料空洞"。
- 不使用 React/Vue/打包器；不引入地图框架（MapLibre/Leaflet/echarts）——一切自绘。
- 除 `data/` 与字体外无外部资源；不加载贴图噪声、HDRI、模型。
- 只用一个主动画循环；UI 全键盘可达、有可访问标签。
- 代码按「uniform / 投影 / 地形 / 疆域 / 城市与旗帜 / 事件引擎 / 路线特效 / UI / 动画循环」清晰分区；非显然的数学写简明注释。
- 产出干净完整的代码：无 TODO、无省略号、无缺失导入、无虚构 API、无省略 CSS。
