## 一、素材存储方案：免费/开源 OSS 对象存储

视频、音频、图片等大文件适合用**对象存储（Object Storage）**，而非传统数据库。

### 1.1 开源对象存储软件（自建）

| 项目 | 特点 | 适用场景 |
|------|------|---------|
| **MinIO** | 兼容 S3 API，高性能，Go 编写，支持分布式 | 最主流方案，替代商业 OSS |
| **Ceph** | 分布式存储平台，对象/块/文件全支持 | 大规模集群，需专业运维 |
| **SeaweedFS** | 轻量级，架构简单，支持多数据中心 | 中小规模，快速部署 |
| **Garage** | 极简 Rust 编写，支持 S3 API | 小型项目或个人使用 |

### 1.2 开源媒体资产管理系统（MAM）

需要**管理**素材（标签、检索、预览）时选用：

| 项目 | 特点 |
|------|------|
| **Piwigo** | 开源图片/视频相册，批量管理、标签、权限 |
| **Immich** | 照片视频备份，AI 人脸识别、时间线浏览 |
| **PhotoPrism** | AI 驱动，自动分类、去重 |
| **Jellyfin** | 媒体服务器，管理本地影视库，类似开源版 Plex |

### 1.3 选型建议

| 需求 | 推荐方案 |
|------|---------|
| 只存文件，自建素材库 | **MinIO** + SQLite/PostgreSQL 记录元数据 |
| 需要管理、标签、检索 | **Immich** 或 **Piwigo** |
| 在线预览、转码、流媒体 | **Jellyfin** 或 **PeerTube** |
| 超大规模、多节点 | **Ceph** 或 **MinIO 集群** |

## 二、视频素材整合：从文案到成片

### 2.1 AI "一键成片" 工具

| 工具 | 操作方式 | 优点 | 缺点 |
|------|---------|------|------|
| **剪映 AI 成片** | 粘贴文案 → 选风格 → 自动配音+配素材+字幕+BGM | 真正一键，中文最好，免费 | 素材库有限，历史类素材不够精准 |
| **一帧秒创** | 输入文案 → 自动分镜、配音、配画面、字幕 | 针对中文短视频优化 | 素材匹配精度一般 |
| **腾讯智影** | 同上 | 国内平台，集成度高 | 部分功能付费 |
| **Pictory / Fliki** | 输入文案或文章链接，AI 自动处理 | 海外工具，功能丰富 | 中文支持弱，大多付费 |

### 2.2 推荐：半自动工作流（质量更高）

历史类视频对**素材准确性**要求极高（不能张冠李戴），推荐：

```
文案 → AI 配音（TTS）→ 人工确认素材 → 自动剪辑合成
```

| 环节 | 推荐工具 |
|------|---------|
| 文案 → 语音 | Edge-TTS（免费）、ChatTTS、讯飞配音、剪映文本朗读 |
| 素材获取 | Pixabay / Pexels / 视觉中国（注意版权） |
| 素材管理 | Immich / MinIO + 自建索引 |
| 自动合成 | **剪映专业版** 或 **MoviePy / FFmpeg** |

### 2.3 剪映专业版高效操作

1. 导入文案，使用「文本朗读」生成旁白
2. 导入背景视频，按旁白时长切割对齐
3. 添加 BGM，开启「自动踩点」卡节奏
4. 导出成品

## 三、FFmpeg 完全指南

### 3.1 是什么

视频处理领域的「瑞士军刀」，命令行工具。几乎所有视频软件（包括剪映导出）底层都依赖它。

**安装：**
```bash
# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg

# Windows：官网下载二进制包，加入环境变量
```

### 3.2 核心命令（针对你的需求）

#### ① 视频 + 旁白音频合成（替换原声）
```bash
ffmpeg -i video.mp4 -i narration.mp3   -c:v copy -map 0:v:0 -map 1:a:0   -shortest output.mp4
```

#### ② 视频 + 旁白 + 背景音乐（BGM 音量 30%）
```bash
ffmpeg -i video.mp4 -i narration.mp3 -i bgm.mp3   -filter_complex "[1:a][2:a]amix=inputs=2:duration=first:weights='1 0.3'[a]"   -map 0:v -map "[a]" -c:v copy -shortest output.mp4
```

#### ③ 加字幕（SRT 烧录到画面）
```bash
ffmpeg -i video.mp4 -vf "subtitles=subtitle.srt" -c:a copy output.mp4
```

#### ④ 拼接多段视频（素材按顺序接起来）

先创建 `filelist.txt`：
```
file 'tang_dynasty.mp4'
file 'song_dynasty.mp4'
file 'ming_dynasty.mp4'
```

执行：
```bash
ffmpeg -f concat -safe 0 -i filelist.txt -c copy output.mp4
```

#### ⑤ 视频截取指定时长
```bash
ffmpeg -i input.mp4 -ss 00:00:10 -t 30 -c copy output.mp4
# 从第10秒开始，截取30秒
```

#### ⑥ 调整分辨率/压缩
```bash
ffmpeg -i input.mp4 -vf "scale=1920:1080" -crf 23 output.mp4
```

### 3.3 FFmpeg 速查表

| 需求 | 关键参数 |
|------|---------|
| 替换音频 | `-i video -i audio -map 0:v -map 1:a` |
| 混音 | `-filter_complex amix=inputs=2` |
| 加字幕 | `-vf subtitles=file.srt` |
| 拼接 | `-f concat -i filelist.txt` |
| 截取 | `-ss 开始时间 -t 时长` |
| 压缩 | `-crf 23`（数值越小质量越高，18-28 常用） |

## 四、MoviePy 完全指南

### 4.1 是什么

基于 FFmpeg 的 Python 库，用代码操作视频，适合写**自动化流水线**。

**安装：**
```bash
pip install moviepy
# 需额外安装 ImageMagick（用于文字渲染）
# macOS: brew install imagemagick
# Ubuntu: sudo apt install imagemagick
```

### 4.2 基础概念

```python
from moviepy import *

# 核心类：
# VideoFileClip("视频文件")      → 视频片段
# AudioFileClip("音频文件")      → 音频片段
# TextClip("文字", fontsize=50)  → 文字片段
# CompositeVideoClip([...])      → 多层叠加
# CompositeAudioClip([...])      → 多音轨混合
# concatenate_videoclips([...])  → 顺序拼接
```

### 4.3 实战脚本 1：基础合成

把视频、旁白、BGM、字幕缝在一起：

```python
from moviepy import *

# 1. 加载素材
video = VideoFileClip("footage.mp4")
voiceover = AudioFileClip("narration.mp3")
bgm = AudioFileClip("bgm.mp3").volumex(0.3)

# 2. 视频匹配旁白时长
if video.duration < voiceover.duration:
    video = video.loop(duration=voiceover.duration)
else:
    video = video.subclip(0, voiceover.duration)

# 3. 音频合成（旁白 + BGM）
final_audio = CompositeAudioClip([voiceover, bgm])
final_video = video.set_audio(final_audio)

# 4. 加字幕（需 srt 文件）
final_video = final_video.subtitles("subtitle.srt")

# 5. 导出
final_video.write_videofile(
    "output.mp4",
    fps=24,
    codec='libx264',
    audio_codec='aac'
)
```

### 4.4 实战脚本 2：多段历史素材拼接

每段素材配一段旁白，自动拼接：

```python
from moviepy import *

segments = [
    {"video": "clips/qin.mp4", "audio": "audio/qin.mp3", "title": "秦朝：大一统"},
    {"video": "clips/han.mp4", "audio": "audio/han.mp3", "title": "汉朝：丝绸之路"},
    {"video": "clips/tang.mp4", "audio": "audio/tang.mp3", "title": "唐朝：盛世长安"},
]

clips = []

for seg in segments:
    v = VideoFileClip(seg["video"])
    a = AudioFileClip(seg["audio"])

    # 视频截取到音频长度
    v = v.subclip(0, min(v.duration, a.duration))

    # 加标题文字
    txt = (TextClip(seg["title"], fontsize=60, color='yellow',
                    font='Source-Han-Sans-CN-Bold')
           .set_position('center')
           .set_duration(2)
           .fadein(0.5).fadeout(0.5))

    segment = CompositeVideoClip([v, txt]).set_audio(a)
    clips.append(segment)

# 拼接所有片段
final = concatenate_videoclips(clips, method="compose")

# 加全局 BGM
bgm = AudioFileClip("bgm.mp3").volumex(0.2).subclip(0, final.duration)
final = final.set_audio(CompositeAudioClip([final.audio, bgm]))

final.write_videofile("history_final.mp4", fps=24)
```

### 4.5 实战脚本 3：批量处理文件夹

自动处理整个素材目录：

```python
from moviepy import *
import os

input_dir = "raw_clips/"
output_dir = "processed/"
os.makedirs(output_dir, exist_ok=True)

for filename in os.listdir(input_dir):
    if filename.endswith(".mp4"):
        filepath = os.path.join(input_dir, filename)
        clip = VideoFileClip(filepath)

        # 统一处理：裁剪、加水印、压缩
        processed = (clip
            .subclip(0, min(clip.duration, 60))  # 最多60秒
            .resize(height=1080)                  # 统一高度
        )

        output_path = os.path.join(output_dir, filename)
        processed.write_videofile(output_path, fps=24, crf=23)
        processed.close()
        clip.close()
```

### 4.6 MoviePy 常用 API 速查

| 操作 | 代码 |
|------|------|
| 加载视频 | `VideoFileClip("file.mp4")` |
| 加载音频 | `AudioFileClip("file.mp3")` |
| 截取片段 | `.subclip(start, end)` |
| 调整音量 | `.volumex(0.5)` |
| 调整尺寸 | `.resize(width=1920)` |
| 循环播放 | `.loop(duration=60)` |
| 淡入淡出 | `.fadein(1).fadeout(1)` |
| 加文字 | `TextClip("文字", fontsize=50).set_position("center")` |
| 视频叠加 | `CompositeVideoClip([bg, fg])` |
| 音频混合 | `CompositeAudioClip([a1, a2])` |
| 顺序拼接 | `concatenate_videoclips([c1, c2])` |
| 设置音频 | `video.set_audio(audio)` |
| 导出 | `.write_videofile("out.mp4", fps=24)` |

## 五、FFmpeg vs MoviePy 选型对比

| 场景 | 推荐 | 原因 |
|------|------|------|
| 临时处理单个文件 | FFmpeg | 一条命令搞定 |
| 批量自动化、复杂逻辑 | MoviePy | Python 代码灵活，可接数据库/AI |
| 实时流媒体/推流 | FFmpeg | 性能更好，延迟更低 |
| 精确到帧的剪辑 | FFmpeg | 时间戳控制更精准 |
| 复杂动画字幕/特效 | MoviePy | TextClip 支持动画效果 |
| 音画同步微调 | 两者皆可 | FFmpeg 更精确，MoviePy 更直观 |

## 六、推荐工作流：历史视频批量生产

### 6.1 完整流水线

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   文案撰写   │ → │  AI 语音合成  │ → │  人工素材匹配 │
│  (分段处理)  │    │ (Edge-TTS等) │    │ (朝代准确性) │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                              │
┌─────────────┐    ┌─────────────┐           │
│   成品输出   │ ← │  自动剪辑合成  │ ←─────────┘
│  (MP4文件)   │    │ (MoviePy脚本) │
└─────────────┘    └─────────────┘
```

### 6.2 各环节工具

| 环节 | 工具/方法 |
|------|----------|
| 文案 | ChatGPT / Claude 辅助，人工校对史实 |
| 语音 | Edge-TTS（免费）、ChatTTS、讯飞配音 |
| 素材 | Pixabay / Pexels / 自制素材 / 购买授权 |
| 存储 | MinIO / 本地 NAS |
| 剪辑 | MoviePy 自动化脚本 |
| 字幕 | Whisper 语音识别生成 SRT |

### 6.3 为什么历史视频必须人工确认素材

AI 自动匹配素材时容易出错：
- 把清朝画面配到明朝文案
- 用影视剧片段当作历史影像
- 人物/建筑/服饰时代错误

**建议**：AI 处理重复劳动（配音、字幕、对齐），你把控核心内容准确性。

## 七、版权合规指南

### 7.1 素材来源

| 来源 | 版权状态 | 使用建议 |
|------|---------|---------|
| **Pixabay** | CC0，可免费商用 | 优先选择 |
| **Pexels** | 免费商用 | 优先选择 |
| **Unsplash** | 免费商用 | 图片为主 |
| **Wikimedia Commons** | 部分公有领域 | 注意查看授权协议 |
| **自制拍摄/录制** | 自有版权 | 最安全 |
| **网易云音乐伴奏** | ⚠️ 有版权 | **不可直接商用** |
| **影视剧片段** | ⚠️ 有版权 | 可能涉及侵权 |

### 7.2 背景音乐（BGM）解决方案

| 方案 | 说明 |
|------|------|
| **YouTube Audio Library** | 免费，部分需署名 |
| **Free Music Archive** | 部分 CC 授权 |
| **Bensound / Mixkit** | 免费 tier 可用 |
| **Epidemic Sound** | 付费，但质量高，适合商用 |
| **AI 生成音乐** | Suno / Udio 生成，注意平台版权政策 |
| **网易云音乐** | ❌ 伴奏也有版权，不能直接用于视频 |

### 7.3 安全使用原则

1. **优先使用 CC0 / 公有领域素材**
2. **商用时查看具体授权协议**（CC-BY 需署名，CC-NC 不可商用）
3. **音乐不要使用平台下载的流行歌曲伴奏**
4. **影视剧片段慎用**，即使是老纪录片也可能有版权
5. **保留素材来源记录**，以备查验

## 八、快速启动清单

### 环境准备
```bash
# 1. 安装 FFmpeg
brew install ffmpeg          # macOS
sudo apt install ffmpeg      # Ubuntu

# 2. 安装 MoviePy
pip install moviepy

# 3. 安装 ImageMagick（文字渲染需要）
brew install imagemagick     # macOS
sudo apt install imagemagick # Ubuntu

# 4. 安装 TTS（可选）
pip install edge-tts
```

### 最小可运行示例

```python
from moviepy import *

# 最简单的合成：视频 + 音频
video = VideoFileClip("input.mp4").subclip(0, 10)
audio = AudioFileClip("voice.mp3").subclip(0, 10)
result = video.set_audio(audio)
result.write_videofile("output.mp4", fps=24)
```
