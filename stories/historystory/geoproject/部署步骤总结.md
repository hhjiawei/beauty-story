# 夏朝故事地图 · 完整部署步骤

## 一、文件功能说明（先搞清楚每个文件是干嘛的）

| 文件 | 功能 | 运行次数 |
|------|------|---------|
| **pipline_pyramid.py** | 数据生产：下载 DEM → 山体阴影 → 高程着色 → 切片 → 生成瓦片金字塔(z=4,5,6) | **一次**（耗时约2-5分钟） |
| **start_server_v2.py** | 服务启动：扫描瓦片层级 → 生成基础 index.html → 启动 HTTP 服务器 → 自动打开浏览器 | **每次看地图前运行** |
| **xia_story.html** | 故事地图前端：3D视角 + 行军箭头动画 + 右侧故事面板 + 时间线联动 | **手动访问**（不需要运行） |

> ⚠️ **重要**：`pipline_pyramid.py` 和 `start_server_v2.py` **功能不重复**，一个负责**生产数据**，一个负责**提供服务**。两者都需要，但分工不同。
>
> 如果你嫌 `start_server_v2.py` 啰嗦，可以用 `python -m http.server 8000` 替代，但少了"自动检测层级+自动打开浏览器"的便利性。

### Step 2：启动 HTTP 服务器（运行 start_server_v2.py）

```bash
cd D:\pycode\GeoTest
.venv\Scripts\python.exe start_server_v2.py
```

**预期输出：**
```
📂 工作目录: D:\pycode\GeoTest
📦 检测到瓦片层级: z=4~6
   z=4: 8 张
   z=5: 30 张
   z=6: 117 张
📝 已生成: D:\pycode\GeoTest\index.html (minZoom=4, maxZoom=6, initialZoom=4)

🚀 服务器启动: http://localhost:8000
🌍 自动打开浏览器...

按 Ctrl+C 停止服务器
```

**这个脚本做了三件事：**
1. **扫描** `tiles/relief/` 目录，自动检测你有 z=4,5,6 哪些层级
2. **生成** `index.html`（基础地图展示页，带正确的 minZoom/maxZoom）
3. **启动** `http://localhost:8000` 服务器
4. **自动**打开浏览器访问

> ⚠️ **不要关掉这个终端**，服务器需要一直运行。需要停止时按 `Ctrl+C`。

### Step 3：访问故事地图（浏览器手动访问）

`start_server_v2.py` 会自动打开浏览器，但它打开的是**基础地图**（`index.html`）。

**你需要手动访问故事地图：**

在浏览器地址栏输入：
```
http://localhost:8000/xia_story.html
```

或者如果之前保存的是 `xia_3d_march.html`：
```
http://localhost:8000/xia_3d_march.html
```

## 三、关于"功能是否重复"的详细判断

### ❌ 不重复，但 start_server_v2.py 可以被简化

| 功能 | pipline_pyramid.py | start_server_v2.py | python -m http.server |
|------|-------------------|-------------------|----------------------|
| 下载 DEM 瓦片 | ✅ | ❌ | ❌ |
| 山体阴影计算 | ✅ | ❌ | ❌ |
| 高程着色 | ✅ | ❌ | ❌ |
| 切片 z=4,5,6 | ✅ | ❌ | ❌ |
| 启动 HTTP 服务 | ❌ | ✅ | ✅ |
| 自动检测层级 | ❌ | ✅ | ❌ |
| 自动生成 index.html | ❌ | ✅ | ❌ |
| 自动打开浏览器 | ❌ | ✅ | ❌ |

**结论：**
- `pipline_pyramid.py` **必须保留**，它是数据生产核心，没有它就没有瓦片。
- `start_server_v2.py` **建议保留**，虽然 `python -m http.server 8000` 也能起服务，但 `start_server_v2.py` 额外提供了：
  - 自动检测层级（防止 404）
  - 自动生成正确的 index.html
  - 自动打开浏览器
  - 对新手更友好

如果你**非常确定**瓦片层级没问题，也可以不用 `start_server_v2.py`，直接：
```bash
python -m http.server 8000
```
然后手动访问 `http://localhost:8000/xia_story.html`。

## 四、日常操作流程（瓦片已生成后）

以后每次想看地图，只需要做**两步**：

```bash
# 1. 启动服务器（在 GeoTest 目录）
cd D:\pycode\GeoTest
.venv\Scripts\python.exe start_server_v2.py

# 2. 浏览器访问（手动输入地址）
# http://localhost:8000/xia_story.html
```

**不需要再运行 `pipline_pyramid.py`**，除非你想：
- 重新生成更高质量的瓦片
- 扩大/缩小地理范围（修改 BOUNDS）
- 调整高程着色颜色（修改 COLOR_STOPS）

## 五、常见问题速查

| 现象 | 原因 | 解决 |
|------|------|------|
| 地图空白/灰色 | 瓦片服务器没启动 | 确认 `start_server_v2.py` 在运行 |
| 404 tiles/relief/4/... | 层级缺失 | 重新运行 `pipline_pyramid.py` |
| 只有地形没有故事面板 | 访问的是 index.html 不是 xia_story.html | 手动访问 `xia_story.html` |
| 箭头不动 | 没点击播放按钮 | 点击底部 ▶ 或按空格键 |
| 视角不能旋转 | 没开启 3D | 检查 xia_story.html 是否有 `pitch: 55` |
| 右键另存为视频 | 需要录屏 | 用 OBS 或 Win+G 录屏 |
