## 一、三个文件到底各管什么？

| 文件 | 角色 | 通俗比喻 | 核心职责 |
|------|------|----------|----------|
| `pipline_pyramid_v2.py` | **工厂** | 印刷厂 | 离线生产瓦片：下载原始高程 → 计算阴影 → 上色 → 切成256×256小图 → 存到 `tiles/` 文件夹 |
| `start_server_v2.py` | **门卫** | 展厅保安 | 启动一个本地HTTP服务器，把 `tiles/` 里的瓦片暴露出去；自动生成一个**2D**查看页 `index.html` |
| `xia_3d_terrain.html` | **展厅** | 策展人 | 真正的3D交互页面，调用瓦片数据，控制视角、倾斜、战役路线动画 |

**关键结论：**
- 地图**长什么样**（颜色、阴影、清晰度）→ 由 **工厂 (`pipline...`)** 决定
- 地图**怎么看**（3D倾斜、旋转、缩放）→ 由 **展厅 (`xia_3d...`)** 决定
- `start_server...` 只管开门，不管装修和展品

## 二、地图样式在哪里控制？

所有「装修参数」都在 `pipline_pyramid_v2.py` 最顶部的**配置区**。你不需要看懂全部代码，只需要知道改这几个数字会发生什么：

| 参数 | 控制什么 | 调大/调小会怎样 |
|------|----------|----------------|
| `COLOR_STOPS` | 海拔对应的颜色 | 决定你的地图是「现代绿黄白」还是「古风赭石褐」 |
| `ELEVATION_STOPS` | 颜色切换的海拔高度 | 比如把 3000 米才变白，改成 1500 米就变白，高山会提前「积雪」 |
| `SEA_COLOR` | 海洋颜色 | 现在偏蓝灰，改成 `[200, 180, 150]` 就是古地图的「淡黄褐海」 |
| `PAPER_COLOR` | 纸张底色 | 现在偏米白，调深一点就是「做旧羊皮纸」效果 |
| `AZIMUTH` | 光源方向（太阳从哪边照） | 现在 315° 是西北光，改成 180° 就是正南光，山脉阴影方向会反转 |
| `ALTITUDE` | 太阳高度角 | 现在 48° 是斜射，阴影较长；改成 80° 接近正午顶光，阴影几乎消失，地形变平 |
| `Z_FACTOR` | 阴影计算时的地形夸张 | **关键参数！** 现在 1.0 比较保守，改成 2.0~3.0 阴影对比度暴增，山脊山谷更锐利 |
| `SHADE_WEIGHT` | 阴影叠加到颜色的强度 | 现在 0.38，改成 0.6 地形立体感更强，但过深会显脏 |
| `WEBP_QUALITY` | 输出图片质量 | 现在 88，调到 95 瓦片更清晰，但文件体积变大 |

**怎么生成你喜欢的古风？**
1. 把 `COLOR_STOPS` 里的现代绿（`[160, 210, 140]` 这种）换成赭石、土黄、灰褐系列
2. 把 `SEA_COLOR` 从蓝灰改成 `[195, 185, 165]`（古海水色）
3. 把 `PAPER_COLOR` 调深一点，增加旧纸感
4. 适当提高 `Z_FACTOR` 到 1.5~2.0，让阴影更强烈（类似古地图的「皴法」）

## 三、清晰度怎么提高？

清晰度分三层，层层加码：

**第一层：输出质量（工厂端）**
- 把 `WEBP_QUALITY` 从 88 提到 95+。这是最简单的，直接让每张瓦片压缩更少、保留更多细节。
- 但注意：Terrarium 原始数据本身只有 **z=6**（约 250 米/像素），这是**数据源的天花板**。无论你输出质量调多高，原始数据就那么多细节。

**第二层：数据源（根本瓶颈）**
- 你现在的 DEM 来自 `Terrarium`（Mapzen/AWS），免费且全球覆盖，但最高只到 z=6 左右。
- 如果要**真正的高清**，需要换更高精度的数据源，比如：
  - **NASA SRTM**（30米精度）
  - **ALOS World 3D**（30米）
  - **Copernicus DEM**（30米）
- 这些数据的 3D 起伏会更细腻，但下载量和计算量会大几十倍。

**第三层：前端显示（展厅端）**
- `xia_3d_terrain.html` 里 `tileSize: 256` 是标准尺寸。在高分屏（Retina）上，MapLibre 默认只请求 256，你可以通过 CSS 或 MapLibre 的 `pixelRatio` 设置让屏幕以 512 逻辑像素渲染，视觉上更锐。
- 另外，你的 `relief` 瓦片是 WebP，如果浏览器解码有模糊，可以尝试输出 PNG（无损）。

## 四、为什么你只感觉到「倾斜」，没感觉到「山脉起伏、盆地凹陷」？

这是你最核心的问题。原因不是代码错了，而是**多个因素叠加导致 3D 感被「吃掉」了**：

### 1. 地形夸张倍数（`exaggeration`）对这片区域来说还是太低
`xia_3d_terrain.html` 里设置了 `exaggeration: 3.5`。这个值在青藏高原（高差 4000 米+）会很震撼，但你的战场在**豫西-晋南**（河南西部到山西南部）：
- 中条山主峰约 2200 米
- 运城盆地约 300-400 米
- 相对高差只有 **1000-2000 米**

在 zoom=5（省级视角）的地图尺度上，2000 米的高差被压缩到屏幕上可能只有几个像素。3.5 倍夸张后变成 7000 米视觉效果——但在整个华北平原的尺度上，依然像一个小土包。

**打个比方：** 你在万米高空看地面，即使把喜马拉雅山拔高 3.5 倍，看起来也只是一条皱纹。

### 2. 你的视角太远（Zoom 级别太低）
`xia_3d_terrain.html` 默认 `zoom: 5.0`，`pitch: 75`。
- **zoom 5** 是「看整个河南省」的距离
- 真正的 3D 起伏感需要 **zoom 8~12**（贴到山体跟前看）才能明显
- 你现在相当于「站在太空舱里看地球」，只能看到角度倾斜，看不到地表褶皱

### 3. 2D 阴影在「欺骗」你的眼睛
`pipline` 生成的 `relief` 瓦片本身就是**带山体阴影的 2D 图片**。人眼看到这个阴影，大脑会自动脑补出「这是立体的」。当 MapLibre 再把这张 2D 阴影图贴到 3D 地形上时，**2D 阴影和 3D 起伏互相干扰**，反而让真正的 3D 变形变得不明显。

**古画比喻：** 你拿一张带皴法的山水画（2D hillshade）贴到一个轻微起伏的石膏模型（3D terrain）上。因为画本身已经很「立体」了，你反而感觉不到石膏的真实凹凸。

### 4. DEM 瓦片可能根本没加载成功（最隐蔽的原因）
`xia_3d_terrain.html` 里有这段代码：
```javascript
fetch(testUrl).then(r => {
  if (!r.ok) throw new Error('DEM 404');
}).catch(e => {
  is3D = false;
  map.setTerrain(null);  // ← 直接关闭3D！
  map.setPitch(0);       // ← 变回2D平铺！
});
```
如果 `tiles/dem/` 目录下缺少文件（比如 z=5 的瓦片没下载全），MapLibre 会**静默降级成 2D**。你看到的「倾斜」可能只是 `pitch: 75` 的 2D 地图倾斜，而不是真正的 3D 地形。

### 5. 数据精度本身就不足以表现「盆地凹陷」
鸣条之战所在的运城盆地，周围的中条山、峨眉台地相对高差约 500-1500 米。在 Terrarium z=6（约 250 米/像素）的数据里，这只有 **2-6 个像素的高差**。即使 exaggeration 拉到 10 倍，在屏幕上也只是一条模糊的边缘，很难形成「盆地凹陷」的视觉冲击力。

## 五、如果想真正看到「山脉起伏、盆地凹陷」，该怎么做？（不改代码的思路）

| 问题 | 解决思路 |
|------|----------|
| **起伏不够** | 把 `exaggeration` 从 3.5 提到 **8~15**。会失真，但视觉冲击力会出来。 |
| **视角太远** | 把默认 `zoom` 从 5.0 改成 **7~8**，把 `pitch` 从 75 改成 **85**（几乎贴地），然后让地图自动飞到战役地点。 |
| **DEM没加载** | 检查 `tiles/dem/` 文件夹里是否真的有 `5/25/12.png` 这类文件。如果没有，说明 `pipline` 的 DEM 下载部分没跑完。 |
| **数据精度不够** | 换更高精度的 DEM 源（30米级），或者**局部放大**——只下载运城盆地周边的高精度区域，而不是整个中国。 |
| **2D阴影干扰** | 做一个「纯高程着色、无阴影」的底图版本，让 3D 地形自己「说话」，而不是靠 2D  hillshade 欺骗视觉。 |

---

## 总结一句话

**你的 3D 没效果，99% 是因为「在太远的地方看太矮的山，用太粗糙的数据，还可能 DEM 根本没加载上」。**

样式问题去 `pipline_pyramid_v2.py` 的配置区调色；3D 问题去 `xia_3d_terrain.html` 检查 DEM 是否存在，并提高 exaggeration + 拉近 zoom。`start_server_v2.py` 只是个开门保安，不用管它。
