## 先秦历史疆域动态可视化方案（定稿）

---

### 一、核心共识

| 决策 | 说明 |
|------|------|
| **时间单位** | 时间切片（关键节点静态版图 + 帧间过渡动画） |
| **空间单位** | 县/都城/关隘（Point）→ Voronoi 势力面（Polygon） |
| **变色逻辑** | 城池得失 → 该城 Voronoi 面变色 → 国家疆域重算 |
| **数据驱动** | JSON 事件流为主，AI 辅助补全模糊坐标，人工主编裁定 |

---

### 二、数据架构：三层表

#### 第一层 `cities.json`（城池基表）
先秦的「国土」本质是城邑控制网络，这是整个系统的锚点。

```json
{
  "cities": [
    {
      "id": "c_001",
      "name": "邯郸",
      "ancient_name": "邯郸",
      "coords": [114.49, 36.60],
      "type": "capital",        // capital | city | fortress | pass
      "state": "赵",            // 初始归属
      "weight": 10,             // Voronoi 权重：都城10，普通城5，关隘8
      "era": "战国",            // 用于筛选
      "note": "赵国都城，前386年迁都至此"
    }
  ]
}
```

**关键字段：**
- `weight`：控制 Voronoi 面大小。都城辐射范围大，普通县城小。
- `type`：决定图标样式（都城用宫殿图标，关隘用城墙图标）。

#### 第二层 `events.json`（事件流）
这是你的「剧本」，一集视频对应一个事件流。

```json
{
  "events": [
    {
      "id": "evt_001",
      "year": "前354年",
      "title": "桂陵之战",
      "era": "战国",
      "attacker": "齐",
      "defender": "魏",
      "sequence": [
        {
          "step": 1,
          "date": "前354年十月",
          "city_id": "c_001",
          "action": "siege",        // siege | capture | retreat | raze | cede
          "result": "魏军围邯郸",
          "new_owner": "魏",        // 围城期间算攻方控制
          "duration": "1年"
        },
        {
          "step": 2,
          "date": "前353年十月",
          "city_id": "c_020",
          "action": "capture",
          "result": "孙膑伏击庞涓于桂陵，齐胜",
          "new_owner": "齐"
        }
      ],
      "territory_snapshot": {
        "before": { "赵": ["c_001", "c_002"], "魏": ["c_003"] },
        "after":  { "赵": ["c_002"], "魏": ["c_001"], "齐": ["c_020"] }
      }
    }
  ]
}
```

**action 类型设计：**
| 类型 | 视觉表现 | 用途 |
|------|----------|------|
| `siege` | 城池图标变橙 + 闪烁 | 围困状态 |
| `capture` | 城池+面变色 | 正式占领 |
| `retreat` | 城池图标变灰 + 箭头回撤 | 战败撤退 |
| `raze` | 城池图标变骷髅 + 面变暗 | 屠城/焚毁 |
| `cede` | 城池平滑变色（无战斗） | 外交割让 |

#### 第三层 `states.json`（国家配色表）
```json
{
  "states": {
    "秦": { "color": "#c45c3e", "fill_opacity": 0.35, "stroke": "#a03020" },
    "赵": { "color": "#2a7d8f", "fill_opacity": 0.35, "stroke": "#1a5d6f" },
    "齐": { "color": "#d4a017", "fill_opacity": 0.35, "stroke": "#b0800f" }
  }
}
```

### 三、可视化层：Voronoi 势力面 + 时间切片

#### 1. 为什么用加权 Voronoi，而不是凸包或手绘边界？

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **手绘边界** | 最精确 | 工作量巨大，先秦无据可考 | ❌ 不可行 |
| **凸包（Convex Hull）** | 计算简单 | 把无人区包进国土，形状虚假 | ❌ 不符合先秦「点状控制」现实 |
| **Voronoi 图** | 每个城只控制最近区域，符合「城邑辐射」逻辑 | 边界生硬，不考虑地形 | ✅ 最优折中 |
| **加权 Voronoi** | 都城控制范围更大，普通城更小 | 计算稍复杂 | ✅ **推荐** |

**加权 Voronoi 原理：**
- 普通县城：权重 5，控制半径约 50km
- 都城：权重 10，控制半径约 100km
- 关隘：权重 8，控制半径约 80km
- 两个城的 Voronoi 边界不是等距线，而是按权重偏移

#### 2. 时间切片渲染流程

```
第 1 帧（战前）：
  1. 读取 cities.json 初始归属
  2. 生成 Voronoi 面
  3. 按 states.json 配色
  4. 叠加地形底图 + 城池图标
  5. 输出：static_001.png

第 2 帧（围城）：
  1. 读取 events.json 第 1 个 step
  2. 邯郸归属改为「魏」，action=siege
  3. 重新生成 Voronoi（邯郸权重临时降低 30%，表示控制力弱）
  4. 邯郸图标变橙 + 闪烁标记
  5. 输出：static_002.png

第 3 帧（战后）：
  1. 读取 events.json 第 2 个 step
  2. 邯郸归属改为「齐」，action=capture
  3. 重新生成 Voronoi
  4. 邯郸图标变红（齐色）+ 插胜利旗帜
  5. 输出：static_003.png

过渡动画（MoviePy）：
  static_001.png → static_002.png：颜色渐变 1.5 秒
  static_002.png → static_003.png：颜色渐变 1.5 秒
```

#### 3. 帧与帧之间的过渡效果

| 效果 | 实现方式 | 用途 |
|------|----------|------|
| **颜色渐变** | MoviePy `crossfade` | 城池易主 |
| **面收缩/扩张** | GeoPandas 逐帧插值 Polygon 顶点 | 疆域进退 |
| **箭头流动** | Matplotlib 画贝塞尔曲线 + MoviePy 逐帧移动 | 军队行进路线 |
| **城池闪烁** | 关键帧透明度 0→1→0 | 围城/激战 |
| **烟雾/火焰** | 叠加 PNG 素材 | 焚城 |

### 四、与现有工具链的衔接

你已有的 Python 生态可以直接复用：

| 现有工具 | 在新方案中的角色 |
|----------|----------------|
| **GeoPandas** | 读取 cities.json → 生成 Voronoi 多边形 → 按 owner 字段着色 |
| **Shapely** | 计算 Voronoi 边界 + 处理城池重叠（两个城太近时合并） |
| **Matplotlib** | 渲染每一帧静态版图（底图 + Voronoi 面 + 城池图标） |
| **MoviePy** | 把静态帧串成视频，加过渡动画、字幕、音效 |
| **Folium** | 快速预览（可选），但正式出片用 Matplotlib 更可控 |
| **CHGIS / 高德 API** | 提供城池坐标底表，AI 补全先秦缺失坐标 |

**新增需要引入的库：**
- `scipy.spatial.Voronoi`：生成加权 Voronoi 图
- `shapely.ops.unary_union`：把同国的 Voronoi 面合并成连续疆域（可选）

### 五、工作流程（从史料到视频）

```
Step 1：史料整理（你 + AI）
  └─ 输入：《史记·赵世家》《战国策》等
  └─ AI 提取：时间、地点、参战方、结果
  └─ 你裁定：哪些城得失了、控制度变化

Step 2：城池坐标维护（你 + AI + CHGIS）
  └─ 有现成坐标：直接录入 cities.json
  └─ 无坐标但有地名：AI 根据史料描述推测（如「桂陵在今河南长垣西北」）
  └─ 完全无考：用相邻已知城做空间插值，或标记为「approximate」

Step 3：事件流编写（你）
  └─ 按时间切片写 events.json
  └─ 每个事件确定：哪些城变色、变什么色、什么 action

Step 4：批量生成版图（Python）
  └─ 脚本读取 events.json
  └─ 对每个时间节点：
      1. 更新 cities.owner
      2. 生成加权 Voronoi
      3. 叠加地形底图 + 城池图标 + 军队箭头
      4. 输出 PNG

Step 5：视频合成（MoviePy）
  └─ 按时间顺序读取 PNG
  └─ 帧间加过渡动画（1-2 秒）
  └─ 叠加字幕（时间、事件名、城池名）
  └─ 叠加音效（战鼓、马蹄、旁白）
  └─ 输出 MP4
```

### 六、先秦特殊处理

#### 1. 县制出现前（夏商周春秋早期）
- 没有「县」的概念，只有「城邑」和「采邑」
- **处理**：以「都城 + 重要城邑」为单位，一个大国可能只有 10-20 个点
- **Voronoi 面会很大**，但这是正确的——先秦大国之间确实有大片无主缓冲带

#### 2. 蛮夷/戎狄区域
- 史料不记其城邑，但它们是实际存在的势力
- **处理**：插入「虚拟控制点」（如「北戎」在燕山一带放一个权重 3 的点），用浅灰色 Voronoi 面表示「非华夏势力」

#### 3. 飞地问题
- 先秦常有「某城属齐，但被赵包围」的飞地
- **处理**：Voronoi 会自然产生飞地面（一个红色孤岛在蓝色海洋中），这反而是先秦政治的真实写照，保留即可

### 七、输出标准（一集视频的数据包）

一个战役（如「长平之战」）的完整数据包：

```
changping_battle/
├── cities.json          # 参战各国城池基表（约 30-50 个城）
├── events.json          # 事件流（约 10-15 个时间切片）
├── states.json          # 秦/赵/韩 配色
├── frames/
│   ├── 001_before.png   # 战前版图
│   ├── 002_siege.png   # 秦军围赵军
│   ├── 003_capture.png # 白起夺空仓岭
│   └── ...
└── output/
    └── changping.mp4    # 合成视频
```
