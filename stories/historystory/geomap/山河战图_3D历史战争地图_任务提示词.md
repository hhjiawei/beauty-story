## 2. 现有素材与接入方式

- 古风渲染瓦片：`tiles/henan_relief`，z=8/9/10，WebP 256px，覆盖 110°E–117°E / 31°N–37°N（河南及周边），赭石/土黄/旧纸风格。以 `raster` source 接入，`tileSize: 256`，`raster-opacity: 0.92`，`raster-fade-duration: 300`。
- DEM 瓦片：`tiles/henan_dem`，Terrarium PNG，z=8/9/10。以 `raster-dem` source（`encoding: "terrarium"`）接入，并：
  1. `map.setTerrain({ source: 'dem', exaggeration: 1.5 })`；
  2. 同一 source 增加 `hillshade` 图层，`hillshade-exaggeration: 0.35`，`hillshade-shadow-color: rgba(90, 70, 50, 0.25)`，与 3D 地形叠加但不喧宾夺主。
- 瓦片覆盖范围之外：最底层放全图 `background` 图层，旧纸色 `#ece4d0`；瓦片 bounds 边缘必须自然淡出，禁止露出灰白网格底或明显边界线。
- `data/geo/rivers.json`：线层，青灰 `#8fa39b`，宽度随 zoom 从 0.6 插值到 2.2，opacity 0.8。
- `data/geo/lakes.json` 与 `data/geo/ocean.json`：面层，填充 `#a7b8ac`，opacity 0.55。
- `data/geo/coastline.json`：线层， `#7d918a`，1px。
- `geoBoundaries-CHN-ADM3_simplified.geojson`：细线层（`#8a7a5c`，0.5px，opacity 0.35），默认隐藏，由图例面板中的"现代县界"开关控制显隐。
- `map.setFog()`：启用雾效，近远范围、古色地平线（`color: #ded3ba`，`horizon-blend: 0.12`，`high-color: #e8dfc8`），使 3D 地形远景融入旧纸天色；若当前 MapLibre 版本支持 `setSky`，同时以能力检测方式启用天空。

## 3. 数据规范

三份 JSON 全部由页面在启动时 `fetch` 加载，代码中禁止硬编码任何城市、势力或事件内容。坐标均为 WGS84 `[经度, 纬度]`。

### 3.1 `data/factions.json`

```json
[
  { "id": "shang", "name": "商", "color": "#a63a2b", "founded": -1600, "fallen": null }
]
```

- `id`：唯一 slug；`name`：显示名；`color`：主色 hex；`founded`/`fallen`：建国/灭亡年份，公元前为负数，`fallen: null` 表示存续。
- 初始内置不少于 8 个势力，配色必须互不混淆，推荐：夏 `#4a5560`（玄灰）、商 `#a63a2b`（朱砂）、葛 `#7a6c4f`、韦 `#5d7052`、顾 `#8c6a3f`、昆吾 `#6b4f6e`、东夷 `#3f6c6b`、周 `#b0762a`（赭金）。

### 3.2 `data/cities.json`

```json
[
  { "id": "bo", "name": "亳", "location": [115.65, 34.45], "owner": "shang", "tier": 1, "note": "商汤之都（今河南商丘北，取通说）" }
]
```

- `owner`：第 1 个事件（seq=1）发生之前的归属，取 factions.json 中的 `id`；无主为 `null`。
- `tier`：1=都城，2=大城，3=方国邑。`isCapitalOf` 不写入此文件——都城身份由 `factions` + 事件状态推导（见 §4）。
- 世界范围 `worldBounds = [107.5, 30.5, 119.5, 38.5]`（西安允许超出瓦片核心区，如丰镐在周原）；城市超出瓦片核心区时校验脚本给出警告而非错误，底图旧纸背景自然承接。
- 初始内置不少于 24 座真实古城，坐标取通说位置并在 `note` 注明今地，至少包括：亳、西亳（偃师）、斟鄩、安邑、鸣条、葛、韦（豕韦）、顾、昆吾、相、庇、奄、殷、朝歌、羑里、孟津、牧野、薛、大彭、三鬷、有娀、桐宫、淮夷（人方）据点、丰、镐。

### 3.3 `data/events.json`（长 JSON：一朝全部事件）

顶层结构：

```json
{
  "meta": { "title": "商朝全史事件库", "dynasty": "商", "version": "1.0.0" },
  "events": [ ]
}
```

单个事件的完整字段：

```json
{
  "id": "shang-0006",
  "seq": 6,
  "year": -1600,
  "yearLabel": "约前1600年",
  "title": "鸣条之战",
  "category": "国际",
  "type": "battle",
  "actors": { "primary": "shang", "target": "xia", "supporters": [] },
  "routes": [
    { "faction": "shang", "from": "bo", "toCity": null, "toPoint": [111.05, 35.05], "waypoints": [] },
    { "faction": "xia", "from": "zhenxun", "toCity": null, "toPoint": [111.05, 35.05], "waypoints": [] }
  ],
  "battle": { "name": "鸣条", "location": [111.05, 35.05], "note": "今山西运城夏县西" },
  "outcomes": [
    { "type": "conquer", "faction": "xia", "by": "shang" },
    { "type": "occupy", "city": "zhenxun", "to": "shang" },
    { "type": "occupy", "city": "anyi", "to": "shang" }
  ],
  "summary": "汤率诸侯联军与夏桀决战于鸣条，桀败奔南巢，夏亡。",
  "quote": { "text": "桀败于有娀之虚，桀奔于鸣条，夏师败绩。", "source": "《史记·殷本纪》" },
  "camera": { "zoom": 7.4, "pitch": 58 }
}
```

字段规则：

- `seq`：从 1 开始的唯一递增整数，全库排序键；`year`：数值（公元前为负），允许多个事件同年（以 `seq` 定序）；`yearLabel`：展示文案（如"约前1300年"）。
- `category`：`"国际"` 或 `"国内"`。
- `type`：事件编排类型，决定播放哪套动画，枚举如下（与 §7 一一对应）：

| type | 含义 | 归类 |
| --- | --- | --- |
| `march` | 行军/征伐（A 城 → B 城，未发生会战或无结果） | 国际 |
| `battle` | 会战（两军各自出发，于 `battle.location` 决战） | 国际 |
| `occupy` | 占领（城市归属权变化，无行军过程） | 国际 |
| `conquer` | 灭国（某势力全部城市归于进攻者） | 国际 |
| `ally` | 联盟（联盟成员变为盟主同色系浅色） | 国际 |
| `rebel` | 反叛（反叛方颜色/城市归属变为目标势力） | 国际 |
| `submit` | 归附（某势力所有城市归于目标势力，无战事） | 国际 |
| `move_capital` | 迁都（都城标记沿路线迁移） | 国内 |
| `succeed` | 继位（新君即位） | 国内 |
| `proclaim` | 称王/建国（君主称号或国号确立） | 国内 |
| `other` | 其他大事（灾异、政变、诛杀等，仅横幅+都城脉冲） | 国内 |

- `routes[]`：行军路线。`from` 为城市 id；终点二选一：`toCity`（城市 id）或 `toPoint`（裸坐标，用于会战点、地图边缘来敌）。`waypoints` 为可选途经点数组；缺省时按 §7.1 自动生成弧线。
- `outcomes[]`：状态变更指令，与 `type` 的动画编排严格分离——`type` 决定"怎么演"，`outcomes` 决定"地图变成什么样"。子类型：`occupy{city,to}`、`conquer{faction,by}`、`ally{leader,members[]}`、`rebel{faction,to}`、`submit{faction,to}`、`move_capital{faction,from,to}`、`ruler_change{faction,ruler,title}`、`proclaim{faction,ruler,title}`。
- `quote`：史料原文与出处，必须出自《史记》《尚书》《竹书纪年》《诗经》等可信文献；无把握时省略整个字段，严禁编造引文。
- `camera`：可选，覆盖自动取景参数。

### 3.4 `tools/validate_data.py`

仅使用 Python 标准库。校验：JSON 可解析；必填字段齐备；`seq` 唯一递增；所有城市/势力引用存在；坐标落在 `worldBounds` 内；`type` 与 `category` 归类一致（`battle` 不得标"国内"等）；`type` 所需结构齐备（`battle` 必须有 `battle.location` 与 ≥2 条 routes，`move_capital` 的 outcome 两端城市存在等）；年份大体非递减（乱序给警告）。输出表格式报告，存在错误时以非零码退出。`python tools/validate_data.py` 必须 0 错误通过，代码与数据才算交付。

## 4. 状态机与播放模型（核心架构）

事件播放的本质是"确定状态 + 过渡动画"：

1. **状态推导为纯函数**：`buildState(uptoSeq)` 从 `cities.json` 初始归属出发，依次应用 `seq ≤ uptoSeq` 的全部 `outcomes`，返回不可变状态：`{ cityOwner: Map<cityId, factionId>, factionAlive: Set, capitals: Map<factionId, cityId>, alliances: Map<leaderId, memberId[]>, rulers: Map<factionId, {name, title}> }`。事件总数 ≤500 时每次重建必须小于 10ms。
2. **点选事件 i**：先无动画瞬时重建到 `seq = i-1` 的状态（城市颜色、存亡、都城、图例一次到位），随后播放事件 i 的编排动画，动画末尾将 `outcomes` 以渐变动效落到地图上，最后状态推进到 i。
3. **拖动时间轴回退**：直接重建到目标 seq，不反向播放动画。
4. **自动连播**：从当前 seq 起，依次播放 i、i+1……直到末尾；每个事件播完后停顿 600ms 再进下一个；任何时刻可暂停，暂停即停在当前事件完成态。
5. **动画管理器**：统一时钟（`performance.now`），所有编排由命名的步骤函数组成（Promise 链），注册在 `TYPE_ANIMATIONS` 映射中，键即 `type` 枚举。全部时长乘以 `1 / speed`（speed ∈ 0.5/1/2/4）。切事件、拖进度、关页面时必须能取消进行中的补间，禁止幽灵动画。全部瞬时特效（脉冲环、冲击波、军队标记）汇入同一个 `effects` GeoJSON source，每帧最多 `setData` 一次。
6. **城市易主动效**（所有占领类 outcome 共用的基础动效）：城市填充色在旧主色与新主色之间做 900ms RGB 插值（ease-in-out），同时从城心扩散 2 道同心脉冲环（半径 0→0.35°，透明度 0.7→0，错开 240ms），城徽做 1→1.3→1 的回弹缩放；光晕层随后以新主色重新生长。禁止瞬间硬切颜色。

## 5. 场景搭建与古风底图

- 样式不由服务器 style.json 提供，而是在 `main.js` 中以代码组装完整 style 对象（version 8），图层自下而上：`background`（旧纸）→ `relief` 古风瓦片 → `hillshade` → `ocean/lakes` → `rivers` → `coastline` → `adm3`（默认隐藏）→ 势力范围面 → 城市光晕 → 行军路线层组 → 特效层 → 城市标记层组 → 特效符号。
- 城市标记：用离屏 canvas 生成雪碧图——tier1 都城为五角星城徽、tier2 为方形城垛、tier3 为圆点，填充为势力主色，描边墨色 `#2b241c` 1.5px；`symbol` 图层按 tier 分级尺寸（18/14/10px），随 zoom 插值放大。城市光晕为 `circle` 层，`circle-blur: 1`，半径按 tier 插值（zoom 6 时 26/18/12px），颜色为势力主色、opacity 0.28。
- 城市名称使用独立的 HTML 标签层（`pointer-events: none` 的绝对定位 div，Noto Serif SC 600，墨色，白描边阴影）：每帧用 `map.project` 批量更新 transform，超屏标签跳过；仅当 zoom ≥ 6.2 显示 tier1，≥ 7 显示 tier2，≥ 8 显示 tier3。禁止依赖在线 glyph PBF 渲染中文。
- 势力范围面（图例可开关，默认开）：对每个存活势力，取其所辖城市坐标 `turf.convex` 后 `turf.buffer` 0.6°，再用 `turf.bezierSpline` 平滑；填充势力主色 opacity 0.08，描边同色系 1px 虚线 opacity 0.4。状态变化时以 1s 淡入淡出重建。城市数 < 3 的势力退化为大半径光晕。

## 6. 势力配色系统

- 提供 `lerpColor(hexA, hexB, t)` 与 `mixWithWhite(hex, ratio)` 工具函数（纯 JS，无依赖）。
- 联盟浅色规则：盟主保持主色；第 i 个盟友（i 从 1 起）的全部城市与图例色块变为 `mixWithWhite(盟主主色, min(0.38 + 0.07 * i, 0.62))`——例如商 `#a63a2b` 的第一个盟友约为浅朱砂。图例中盟友缩进排列在盟主之下，以细线连接。
- 灭亡势力：城市转为新主色后，图例色块置灰 `#9a917e` 并加"亡"角标；`fallen` 写回运行时状态。
- 反叛：反叛方城市与图例直接采用目标势力主色（如同其一支），图例中标注"叛"。

## 7. 动画系统详规（每类事件的编排剧本）

以下时长均为 1 倍速基准；缓动除注明外为 ease-in-out。相机行为见 §9，动画期间用户可自由拖拽缩放（仅打断镜头补间，不打断事件动画）。

### 7.1 行军 `march`（A 城 → B 城）

- 路线生成：有 `waypoints` 则串联为平滑折线（`turf.bezierSpline`）；否则自动生成二次贝塞尔弧线——起止点连线中点沿垂直方向偏移距离的 12%，64 段采样。
- 路线"生长"：路线层 paint 使用 `line-gradient`（基于 `line-progress`），每帧用 `map.setPaintProperty` 推进渐变阈值从 0→1，时长 1.8s，颜色为行军势力主色，线宽 4px + 墨色衬线 5.5px；如出现渲染异常，回退方案为按进度切片坐标 `setData`。
- 军队标记：军旗 sprite（canvas 绘制：三角旗 + 主色）沿路径以 `turf.along` 逐帧移动，时长 2.2s，位置用 `map.queryTerrainElevation` 贴地并加微小起伏；尾部拖一段 12% 路径长的半透明残影。
- 到达：终点城市一次脉冲环 + 城徽回弹；若有 `occupy` outcome，衔接 §4-6 的易主动效。

### 7.2 会战 `battle`（两军合聚）

- 第一阶段：≥2 条路线同时生长，各军标记的行进时长统一归一化为 2.4s，保证**同时抵达**会战点；路线颜色各用本方主色。
- 第二阶段（接战）：在 `battle.location` 依次炸开 3 道冲击波环（间隔 240ms，主色取进攻方），浮现"交叉兵刃"sprite，镜头容器 CSS 抖动 6 帧（振幅 8px 衰减），屏幕边缘极轻微朱砂色渐晕闪一次。
- 第三阶段：弹出战果横幅（事件标题 + 一句 summary）；随后依次执行 `outcomes` 对应的落地动效（占领/灭国/归附……）。
- 全程约 4.5s（1 倍速）。

### 7.3 占领 `occupy`：直接执行 §4-6 易主动效；横幅显示"×军取 ×城"。

### 7.4 灭国 `conquer`

- 将灭亡方现存全部城市按到其都城的距离升序排序，以 260ms 间隔级联执行易主动效；都城最后，附加一道更大的玄色（`#2b241c`）烟尘环与 1.4 倍脉冲。
- 全部完成后：图例置灰加"亡"；屏幕下方横幅"夏亡于商"式标题 + `quote` 引文（竖排引文样式，Noto Serif SC 900）；势力范围面淡出。

### 7.5 联盟 `ally`

- 各成员城市颜色 800ms 渐变为 §6 的盟友浅色；盟主都城与各成员都城之间绘制"盟约线"（浅色虚线，1.2s 生长），动画后保留 opacity 0.25 直至联盟状态改变。
- 图例重组为联盟组；横幅"×与×会盟"。

### 7.6 反叛 `rebel`

- 反叛方城市在旧主色与目标主色之间快速闪烁 3 次（每次 160ms）后定格为目标色，象征阵前倒戈；附加一道撕裂状红色脉冲环。
- 图例即时换色并标"叛"；牧野之战等事件中该 outcome 嵌在 `battle` 第三阶段执行。

### 7.7 归附 `submit`

- 无战事动效：归附方全部城市按距目标方都城由近及远，200ms 级联、1.2s 缓变至目标主色；光晕同步过渡；横幅"×归于×"。

### 7.8 迁都 `move_capital`

- 旧都城星徽沿自动生成的弧线移动至新都城（2.4s，贴地），旧都城标记降级为 tier2 城垛（星→城渐变缩放）；到达时朱砂印章"都"字从新都城上方盖下（scale 2.2→1，120ms 回弹，透明度 0→0.9），星徽亮起。
- 横幅"迁于×"；图例都城标记更新。

### 7.9 继位 `succeed`

- 该势力都城两轮柔和金色脉冲（`#d8b45a`）；详情卡展示新君名号与 `quote`。

### 7.10 称王/建国 `proclaim`

- 都城升起主色王旗 sprite（旗面书势力名），一道主色圆环扩展至 0.5°；若为建国，势力范围面首次生长；横幅"×王 ××"（如"汤王于亳"）。

### 7.11 其他 `other`

- 相关都城/城市一次墨色脉冲；详情卡 + `quote`；无地图颜色变化。

## 8. 覆盖层界面与视觉设计

全屏无滚动，地图 canvas 置于最底层，UI 均为玻璃/宣纸质感覆盖层，首个渲染帧完成后统一淡入。整体美术：宣纸底（`rgba(246, 240, 226, 0.88)` + `backdrop-filter: blur(14px)`）、墨色文字 `#2b241c`、朱砂强调 `#a63a2b`、赭金点缀 `#b0762a`、圆角 14px、1px 半透明墨线边框。禁止遮挡地图拖拽：除面板本身外覆盖层 `pointer-events: none`。

1. **事件列表面板（右侧，320px）**：卷轴式编年列表，按 `yearLabel` 分组（吸顶年号头）；每条 = seq 序号 + 类型徽标（每类一个独立小图标与颜色：行军赭、会战朱、占领赤、灭国玄、联盟青、反叛紫、归附绿、迁都金、继位黄、称王红）+ 标题 + 年份；顶部搜索框（按标题/城市/势力过滤）与类型筛选 chips 行 + 势力下拉筛选；点击任意条目 → 按 §4-2 跳转并播放；当前事件条目高亮朱砂描边。
2. **底部时间轴**：播放/暂停、上一事件、下一事件按钮；倍速切换 0.5/1/2/4；进度条上按 seq 等距渲染事件刻度（国际朱砂点、国内金点，灭国为大方点），可点击跳转；左侧大号纪年显示（当前 `yearLabel`）与"第 i / N 事"计数；自动连播开关（默认手动模式）。
3. **事件详情卡（左下，动画期间浮出）**：标题、纪年、类型徽标、参战方 chips（势力主色底）、summary、`quote`（竖排或引文块，注明出处）、相关城市列表（点击 flyTo 该城）；卡片 800ms 滑入，事件播完 2.5s 后滑出。
4. **图例（左下，可折叠）**：势力色块 + 名称 + 都城名；联盟分组缩进；亡势力置灰；下方三个开关：势力范围 / 现代县界 / 城市标签。
5. **顶部标题区（左上）**：眉题"商 · 全史战图"（小字大写式字距），主标题"山河战图"（Noto Serif SC 900，36px），副题"行军 · 会战 · 兴亡 一图演绎"。
6. **加载界面**：旧纸底、居中朱砂印章式"山河战图"方印（SVG 描边动画）、状态文字"山河铺陈中……"；首个 `idle` 事件后 0.8s 淡出。
7. **键盘**：空格=播放/暂停，←/→=上/下一事件。
8. **移动端（max-width 768px）**：列表面板改为底部抽屉（默认收起，上拉展开）；时间轴压缩为两行；详情卡全宽；保留双指缩放与拖拽。

## 9. 相机系统

- 初始：中心 `[113.6, 34.8]`，zoom 6.3，pitch 52，bearing -12；minZoom 5.5，maxZoom 10.5；maxBounds 为瓦片范围四向各外扩 1.5°（周原方向外扩至 worldBounds）。
- 事件取景：对事件涉及的全部坐标（routes 起讫、battle.location、outcome 城市）`fitBounds`，padding 为视口短边 18%，maxZoom 8.2，pitch 补间至 58，时长 1.4s（被事件的 `camera` 字段覆盖）；镜头先到位，动画再开始。
- 会战接战瞬间 zoom 额外推近 0.35，战果横幅后缓回。
- 用户任意拖拽/缩放即取消镜头补间（事件动画继续）。

## 10. 加载与错误状态

- `Promise.all` 加载三份 JSON 与 GeoJSON；任一失败：保持加载底，转为错误界面，标题"山河未展"，列出失败文件与 HTTP 状态；瓦片 404 不阻塞（警告即可，旧纸底承接）。
- 数据校验在浏览器端同步执行与 `validate_data.py` 相同的规则子集；发现错误时弹出错误面板，逐条列出（文件、事件 id、规则），页面保持可用但阻止播放。
- WebGL 不可用：错误界面说明并建议现代浏览器。

## 11. 性能与工程约束

- 单一 `requestAnimationFrame` 主循环：每帧统一推进补间、批量更新 `armies`/`effects` source（各最多一次 `setData`）、批量更新 HTML 标签 transform；300 城市规模下桌面端稳定 60fps。
- 禁止每帧分配大对象（路线、雪碧、样式对象全部预计算复用）；切事件时清理全部进行中的补间与临时图层数据。
- 连续自动播放 50 个事件后无内存持续增长、无图层残留。
- 不得引入 React/Vue 等框架；不得使用视频、GIF、位图底图替代矢量动画；除 §1 指定 CDN 外无其他依赖。
- JS 按"配置 / 工具 / 数据加载 / 状态机 / 图层搭建 / 配色 / 动画（每类一节）/ 相机 / UI / 主循环"分节组织；非直观的几何与插值计算写简洁注释。
- 产出干净完整的代码：无 TODO、无省略、无虚构 API。
