三级热点监测体系：
层级 监测平台 响应时效 适用选题
一级（突发） 微博热搜、微信指数、新榜、抖音热榜 2小时内 重大社会事件、突发新闻

选题评估四维打分表：
维度 权重 评分标准（1-5分）

争议性 25% 观点分歧程度、站队可能性
社会价值 30% 民生关联度、公共议题重要性

选题立项标准：总分≥15分且社会价值≥3分方可立项

步骤2：信源交叉验证
交付物：信源交叉验证表（≥2独立权威源）、时间线、争议点标注
信源分级体系：
等级 信源类型 可信度 使用规范
S级 政府通报、官方声明、权威媒体（新华社/央视/人民日报） 100% 可直接引用，无需标注
A级 主流媒体（澎湃/财新/南方周末）、学术期刊 90% 标注来源，优先采用
B级 当事人公开声明、目击者证词（有证据） 70% 需多方印证，标注"据XX表示"
C级 自媒体爆料、网友反映、匿名信源 50% 必须标注"尚待核实"
交叉验证原则：
核心事实必须≥2个独立S/A级信源确认
时间线必须精确到小时（突发）或天（持续）
争议点必须呈现多方观点，不预设立场

步骤5：正文写作
交付物：初稿（按结构模板）+ 情绪校准说明
5.1 字数控制标准
文章类型 推荐字数 适用场景 阅读完成率
快讯类 300-800字 突发新闻、事件进展通报 90%以上
观点短评 800-1500字 热点事件快速评论、单一角度解读 70%-80%
深度分析 1500-3000字 事件复盘、多维度解读、背景调查 40%-60%
特稿/调查 3000-5000字 重大事件深度调查、人物专访 20%-30%
禁忌：避免无意义凑字数，宁短勿长；超过3000字必须分章节，每1000字左右插入小标题或图片。
5.2 核心写作原则
真实性第一：所有事实必须有可靠来源，注明出处
价值观正确：坚守人道主义立场，同情弱者，追求公平正义
客观中立：避免极端情绪化表达，呈现多方观点
有温度有态度：在客观报道基础上，表达合理情感和鲜明观点
保护隐私：不泄露受害者个人信息，不公开未成年人照片和姓名
5.3 笔风校准口诀
> \*\*"事实要硬，情绪要准；立场要明，表达要稳；批判要深，落点要暖。"\*\*

6.1 黄金排版标准
字体与字号：
正文：14-15号字，行间距1.5-1.75倍，字间距0.5-1px
小标题：16-18号字，加粗，颜色与正文区分（推荐#333333正文，#111111标题）

左右边距15-20px，避免文字贴边

六、标题创作：爆款的第一要素
6.1 社会事件类标题黄金公式
公式1：悬念式
> \*\*结构\*\*：事件+反常点+疑问  
> \*\*示例\*\*："那个捐了100万的清洁工，为什么被全网骂？"
> 公式2：冲突式
> \*\*结构\*\*：对立双方+核心矛盾  
> \*\*示例\*\*："业主花500万买的学区房，却被教育局告知不能入学"
> 公式3：数字式
> \*\*结构\*\*：具体数字+冲击力  
> \*\*示例\*\*："3天2夜，12个家庭的崩溃：这场暴雨暴露了多少城市漏洞"
> 公式4：观点式
> \*\*结构\*\*：鲜明立场+核心结论  
> \*\*示例\*\*："别再骂那个逃跑的老师了，她只是个普通人"
> 公式5：共情式
> \*\*结构\*\*：身份标签+共同经历  
> \*\*示例\*\*："作为一个妈妈，我真的不敢看这次幼儿园事件的细节"
> 公式6：热点嫁接式
> \*\*结构\*\*：热点事件+深度洞察  
> \*\*示例\*\*："从董宇辉事件看：职场人拼到最后，拼的是'反内耗能力'"
> 6.2 标题禁忌
> ❌ 避免使用"震惊体""吓尿体""必看""转疯了"等低俗词汇
> ❌ 避免夸大其词、虚假宣传，如"史上最惨""全世界都在看"
> ❌ 避免使用过于专业的术语和生僻字
> ❌ 避免标题过长，控制在15-25字之间，手机一屏能完整显示
> ❌ 避免使用标点符号堆砌，如"！！！""？？？"
> 6.3 爆款标题案例库
> 类型 标题 成功要素

民生类 《3个被低估的冷门技能，让你少走10年弯路》 数字+痛点+利益承诺
职场类 《"躺平"的95后，正在偷偷买房》 冲突反转+身份标签
情感类 《同事问我工资时，我这样回答…》 悬念+场景代入
热点类 《月薪5000的打工人必看！我靠这3个"抠门技巧"，半年攒下3万块》 身份标签+数字+利益承诺
争议类 《别再盲目存钱了！真正会搞钱的人，都在做"负利率投资"》 否定共识+新观点+悬念

七、内容结构与写作技巧
7.1 开头黄金30字
必须在首段完成：
点明核心矛盾或利益承诺
建立与读者的情感连接
设置继续阅读的钩子
开头公式：
> \*\*痛点场景 + 反常识观点\*\*  
> 示例："年轻人为什么反感加班？不是懒，是规则出了问题。"
> 7.2 主体三段式论证
> 事实层：客观呈现事件经过（引用权威信源）
> 分析层：提供独特视角（避免人云亦云）
> 共鸣层：连接读者个人经历（"这也是你的处境吗？"）
> 7.3 结尾情绪升华
> 三种结尾策略：
> 策略 适用场景 示例
> 金句总结 观点类文章    "真正的成熟，是学会在复杂世界里保持独立思考。"
> 开放式提问 争议类文章    "如果是你，会怎么选择？欢迎在评论区分享。"
> 行动号召 公益/倡议类    "转发给关心的人，让更多人看到真相。"
> 7.4 情绪曲线设计
> 一篇爆款文章的情绪曲线应该是：

```
好奇（开头）→ 愤怒/同情（发展）→ 思考（高潮）→ 希望/共鸣（结尾）
```

情绪控制原则：
允许表达关切与共情
禁用"血泪""黑幕""必严惩"等煽动性表述
批判要有建设性落点

八、传播逻辑与运营策略
8.1 微信生态传播引擎
算法偏好：分享率 × 在看率 × 留言互动 > 单纯阅读量
关键动作：
文末预埋"转发语"：提供2-3句可直接复制转发的总结性话术
评论区运营：作者置顶1条高质量补充/提问，引导UGC；2小时内回复前20条留言
发布时间：热点事件发布后2-6小时（情绪峰值期）或次日9:00/12:30/20:30
8.2 内容"社交货币"设计
提供"谈资型结论"：如"这件事的本质不是XX，而是XX"
提供"身份认同标签"：如"致那些不愿沉默的普通人"
提供"可行动建议"：如"普通人能做的3件小事"
8.3 冷启动与推荐机制
进入"看一看"流量池的关键：
前500阅读决定能否进入公域推荐（转发率>5%）
互动数据（评论质量、在看数）影响算法权重
完读率决定后续推荐力度
启动技巧：
建立核心粉丝群（500-1000人），发布前30分钟集中转发
设计"转发话术"，降低分享门槛
及时回复前20条评论（算法识别为"高互动内容"）

十、合规底线与风险管控
10.1 信源原则
优先采用政府通报、主流媒体、学术期刊、当事人公开声明
自媒体爆料需标注"据网友反映/尚待核实"
核心事实必须≥2个独立权威信源确认
10.2 定性原则
不越权司法/行政定性
使用"疑似""通报显示""有观点认为"等限定词
避免绝对化判断（"必然""肯定""绝对"）
10.3 情绪边界
允许表达关切与共情
禁用"血泪""黑幕""必严惩"等煽动性表述
批判要有建设性落点
10.4 隐私保护
不公开未成年人/受害者非必要信息
打码处理截图
尊重当事人隐私权
10.5 监管动态
定期学习《网络信息内容生态治理规定》《互联网新闻信息服务管理规定》
建立"敏感词库+红线事件清单"
设立"合规一票否决权"，审核岗可要求重写或暂缓发布
10.6 危机处理
如发布错误信息，第一时间删除并发布更正声明
如引发负面舆论，及时回应读者质疑，诚恳道歉
建立应急预案，遇到重大危机及时上报相关部门

# 附录：标题创作完全指南

## 一、标题的重要性

> **标题贡献80%的打开率，前3秒决定用户是否点击。**

在微信公众号生态中，标题是文章的第一道门槛。一个好的标题能够在信息流中脱颖而出，吸引用户点击；一个差的标题即使内容再好，也可能被淹没。

### 1.1 标题的核心功能

| 功能       | 说明         | 实现方式      |
|----------|------------|-----------|
| **吸引注意** | 在信息流中脱颖而出  | 反差、数字、悬念  |
| **传递价值** | 让用户知道能获得什么 | 利益承诺、痛点解决 |
| **激发情绪** | 触发点击冲动     | 共情、愤怒、好奇  |
| **筛选受众** | 精准触达目标读者   | 身份标签、场景描述 |

### 1.2 好标题 vs 坏标题

| 维度       | 好标题       | 坏标题       |
|----------|-----------|-----------|
| **信息密度** | 一句话说清核心价值 | 空洞无物，不知所云 |
| **情绪触发** | 激发好奇或共鸣   | 平淡无奇，无感   |
| **受众精准** | 目标读者一眼识别  | 泛化，无法定位   |
| **传播友好** | 适合转发分享    | 尴尬，不愿转发   |

## 二、社会事件类标题六大黄金公式

### 公式1：悬念式

> **结构**：事件+反常点+疑问

**原理**：打破常规认知，制造认知冲突，激发好奇心。

**适用场景**：

- 有反常识元素的事件
- 结果出人意料的故事
- 存在争议的话题

**案例**：

- "那个捐了100万的清洁工，为什么被全网骂？"
- "月薪3万的程序员，为什么住在城中村？"
- "名校毕业的她，为什么选择去送外卖？"

**创作要点**：

1. 找出事件中最反常的元素
2. 用疑问句式引发好奇
3. 避免直接给出答案

---

### 公式2：冲突式

> **结构**：对立双方+核心矛盾

**原理**：呈现利益冲突或观点对立，激发读者的站队欲望。

**适用场景**：

- 涉及多方利益的事件
- 存在明显对立的争议话题
- 维权/纠纷类事件

**案例**：

- "业主花500万买的学区房，却被教育局告知不能入学"
- "员工加班猝死，公司却说'自愿加班'"
- "老人摔倒被扶，却反讹救助者10万"

**创作要点**：

1. 明确对立双方
2. 突出核心矛盾点
3. 用"却""但是"等转折词强化冲突

---

### 公式3：数字式

> **结构**：具体数字+冲击力

**原理**：数字具有天然的吸引力，具体数字比模糊描述更有说服力。

**适用场景**：

- 涉及数据的事件
- 时间跨度长的故事
- 数量/金额突出的案例

**案例**：

- "3天2夜，12个家庭的崩溃：这场暴雨暴露了多少城市漏洞"
- "35岁，被裁员，房贷200万：一个中产家庭的崩塌"
- "工作10年，存款为0：90后的财务困境"

**创作要点**：

1. 使用具体数字，避免"很多""大量"等模糊词
2. 数字要有冲击力（大或小到极致）
3. 数字与主题强相关

---

### 公式4：观点式

> **结构**：鲜明立场+核心结论

**原理**：直接表达观点，吸引认同者，引发讨论。

**适用场景**：

- 评论类文章
- 有明确立场的话题
- 反主流观点的表达

**案例**：

- "别再骂那个逃跑的老师了，她只是个普通人"
- "996不是福报，是违法"
- "躺平不是消极，是理性的选择"

**创作要点**：

1. 立场要鲜明，避免模棱两可
2. 观点要有争议性或反常识性
3. 准备好支撑观点的论据

---

### 公式5：共情式

> **结构**：身份标签+共同经历

**原理**：用身份标签精准触达目标读者，用共同经历引发共鸣。

**适用场景**：

- 面向特定群体的内容
- 情感类、职场类话题
- 需要建立信任的文章

**案例**：

- "作为一个妈妈，我真的不敢看这次幼儿园事件的细节"
- "35岁的打工人，你累了吗？"
- "北漂第5年，我终于明白了这些道理"

**创作要点**：

1. 身份标签要精准（年龄/职业/地域/状态）
2. 共同经历要有普遍性
3. 语气要真诚，避免说教

---

### 公式6：热点嫁接式

> **结构**：热点事件+深度洞察

**原理**：借助热点流量，提供独特视角，实现差异化竞争。

**适用场景**：

- 热点事件评论
- 需要快速发布的时效性内容
- 借热点讲道理的软文

**案例**：

- "从董宇辉事件看：职场人拼到最后，拼的是'反内耗能力'"
- "从俄乌冲突看普通人的生存智慧"
- "从谷爱凌的成功看家庭教育的真相"

**创作要点**：

1. 热点要新，最好在48小时内
2. 洞察要独特，避免人云亦云
3. 关联要自然，不要硬凑

## 三、标题创作的10个技巧

### 技巧1：使用身份标签

用具体的身份标签精准触达目标读者。

| 通用表述 | 优化后     |
|------|---------|
| 很多人  | 35岁打工人  |
| 年轻人  | 95后职场新人 |
| 家长   | 海淀妈妈    |
| 创业者  | 县城青年    |

### 技巧2：加入时间元素

时间元素增加紧迫感和真实感。

| 原表述   | 优化后             |
|-------|-----------------|
| 我辞职了  | 昨天，我辞职了         |
| 他成功了  | 3年，从月薪3000到年薪百万 |
| 问题很严重 | 这个问题，已经困扰我们10年了 |

### 技巧3：使用对比反差

对比产生张力，反差引发好奇。

| 原表述    | 优化后               |
|--------|-------------------|
| 他很穷    | 月薪3000，却捐了100万    |
| 她很努力   | 每天工作16小时，却被裁员了    |
| 这个问题很难 | 清华教授解不开的题，被小学生破解了 |

### 技巧4：制造悬念缺口

不说完，留一半，激发点击欲望。

| 原表述     | 优化后               |
|---------|-------------------|
| 如何提高效率  | 提高效率的3个方法，最后一个太绝了 |
| 他为什么成功  | 他从负债到身家过亿，只做对了一件事 |
| 这个习惯很重要 | 这个习惯，决定了你90%的人生   |

### 技巧5：使用情绪词汇

情绪词汇直接触发情感反应。

| 中性表述 | 情绪化表述    |
|------|----------|
| 问题   | 崩溃/崩塌/绝望 |
| 成功   | 逆袭/翻盘/封神 |
| 困难   | 噩梦/深渊/绝境 |
| 感动   | 泪目/破防/戳心 |

### 技巧6：借用权威背书

权威增加可信度。

| 原表述     | 优化后            |
|---------|----------------|
| 这个方法很有效 | 哈佛研究表明：这个方法最有效 |
| 专家说     | 钟南山院士提醒：这件事要注意 |
| 很多人不知道  | 90%的人都不知道这个秘密  |

### 技巧7：使用第二人称

"你"字拉近距离，增强代入感。

| 原表述      | 优化后           |
|----------|---------------|
| 职场人要注意   | 你，正在被职场淘汰     |
| 这个方法很有用  | 这个方法，能帮你省下10万 |
| 很多人有这个问题 | 你也有这个问题吗？     |

### 技巧8：承诺具体利益

明确的利益承诺提高点击动力。

| 原表述  | 优化后             |
|------|-----------------|
| 教你省钱 | 3个技巧，帮你一年省下2万   |
| 提高效率 | 这个方法，让我效率提升300% |
| 学会理财 | 理财小白如何月入过万？     |

### 技巧9：使用否定句式

否定句式打破认知，引发好奇。

| 原表述 | 优化后        |
|-----|------------|
| 要努力 | 别再盲目努力了    |
| 要存钱 | 别再盲目存钱了    |
| 要买房 | 年轻人，先别急着买房 |

### 技巧10：控制字数

标题过长会被折叠，控制在24-30字之间。

| 过长                        | 优化后               |
|---------------------------|-------------------|
| 关于如何提高工作效率的10个方法，最后一个太有用了 | 提高效率的10个方法，最后一个绝了 |
| 一个35岁的程序员被裁员之后的真实经历和感悟    | 35岁被裁员后，我才明白这些道理  |

## 四、标题禁忌

### 4.1 绝对禁用

| 类型       | 示例           | 原因        |
|----------|--------------|-----------|
| **震惊体**  | 震惊！吓尿了！转疯了！  | 低俗，降低账号格调 |
| **标题党**  | 点进来有惊喜！不看后悔！ | 欺骗用户，损害信任 |
| **夸大其词** | 史上最惨！全世界都在看！ | 虚假，引发反感   |
| **标点堆砌** | ！！！？？？……     | 不专业，影响阅读  |

### 4.2 谨慎使用

| 类型        | 示例         | 注意事项   |
|-----------|------------|--------|
| **绝对化词汇** | 最、第一、唯一、绝对 | 需有数据支撑 |
| **煽动性词汇** | 血泪、黑幕、必严惩  | 可能触碰红线 |
| **隐私信息**  | 真实姓名、身份证号  | 需脱敏处理  |
| **未经证实**  | 疑似、可能、据说   | 需标注来源  |

## 五、标题A/B测试方法

### 5.1 测试流程

```
创作阶段：写出3-5个备选标题
    ↓
内部评估：团队投票+打分
    ↓
小范围测试：粉丝群/朋友圈测试
    ↓
数据分析：点击率、互动率对比
    ↓
最终确定：选择数据最好的标题
```

### 5.2 评估维度

| 维度  | 权重  | 评分标准        |
|-----|-----|-------------|
| 吸引力 | 30% | 是否能引起点击欲望？  |
| 准确性 | 25% | 是否准确反映正文内容？ |
| 传播性 | 25% | 是否适合转发分享？   |
| 安全性 | 20% | 是否存在风险？     |

### 5.3 测试工具

- **公众号后台**：多标题功能（部分账号开放）
- **粉丝群测试**：在核心粉丝群发布，统计反馈
- **朋友圈测试**：用小号发布，观察互动情况

## 九、总结

### 标题创作的核心心法

> **"一句话说清价值，一个词触发情绪，一个标签精准触达。"**

### 好标题的三要素

1. **信息价值**：读者能获得什么？
2. **情绪价值**：能触发什么情感？
3. **社交价值**：转出去是否有面子？

### 标题创作的终极原则

- **真实**：不标题党，不欺骗用户
- **精准**：一句话说清核心价值
- **克制**：不过度煽情，不触碰红线

## mcp_tool.py中
"""
MCP 连接封装层。
负责：
- 连接 trendradar MCP 服务器
- 将 MCP 工具转换为 LangChain 标准工具
- 管理连接生命周期
"""

import asyncio
from contextlib import asynccontextmanager
from langchain_mcp_adapters.client import MultiServerMCPClient
from langchain_core.tools import BaseTool


class TrendRadarMCPManager:
    """
    trendradar MCP 服务器管理器。
    使用单例模式避免重复连接，支持 async with 上下文管理。
    """

    _instance = None
    _lock = asyncio.Lock()

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._client = None
            cls._instance._tools = None
        return cls._instance

    async def _ensure_connected(self):
        """懒连接：只在第一次获取工具时建立连接"""
        if self._client is not None:
            return

        self._client = MultiServerMCPClient({
            "trendradar": {
                "transport": "streamable_http",
                "url": "http://localhost:3333/mcp"
                # 如果需要认证，加 headers：
                # "headers": {"Authorization": "Bearer YOUR_TOKEN"}
            }
        })
        # 显式连接（某些版本需要）
        # await self._client.connect()

    async def get_tools(self) -> list[BaseTool]:
        """获取 trendradar 的所有 MCP 工具（已转换为 LangChain 格式）"""
        await self._ensure_connected()
        if self._tools is None:
            self._tools = await self._client.get_tools()
        return self._tools

    def get_tools_sync(self) -> list[BaseTool]:
        """同步获取工具（供同步代码调用）"""
        return asyncio.run(self.get_tools())

    async def disconnect(self):
        """清理 MCP 连接"""
        if self._client is not None:
            # MultiServerMCPClient 支持 async with，这里手动关闭
            await self._client.__aexit__(None, None, None)
            self._client = None
            self._tools = None

    @asynccontextmanager
    async def session(self):
        """上下文管理器：自动连接和断开"""
        try:
            tools = await self.get_tools()
            yield tools
        finally:
            await self.disconnect()

@tool
def create_file(file_name: str, file_contents: str) -> str:
    """Create a text file."""
    try:
        file_path = os.path.join(os.getcwd(), file_name)
        os.makedirs(os.path.dirname(file_path), exist_ok=True)
        with open(file_path, "w", encoding="utf-8") as f:
            f.write(file_contents)
        return f"Created: {file_path}"
    except Exception as e:
        return f"Error: {e}"


@tool
def shell_exec(command: str) -> str:
    """Windows CMD/PowerShell executor (only tool for open browser/screenshot)."""
    blacklist = ["del", "rm", "format", "rd"]
    if any(x in command.lower() for x in blacklist):
        return "❌ Blocked dangerous command"
    try:
        res = subprocess.run(command, shell=True, capture_output=True, timeout=20, text=False)
        stdout = res.stdout.decode("gbk", errors="replace")
        stderr = res.stderr.decode("gbk", errors="replace")
        return f"OUT:\n{stdout}\nIN:\n{stderr}".strip()
    except Exception as e:
        return f"Exec error: {e}"
